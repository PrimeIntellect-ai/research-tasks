As a backup administrator, I need to migrate our legacy archival data retrieval system to a modern web service. Our old system used a proprietary tool to archive directories into a custom binary blob and an index file. 

We have lost the source code for the archiving tool, but a stripped compiled version is available at `/app/legacy_archiver`. You can use this binary to pack directories and analyze its output to understand the format (e.g., `mkdir test_dir && echo "hello" > test_dir/a.txt && /app/legacy_archiver test_dir test_data.dat test_index.csv`).

Your objective is to write a C++ HTTP service that serves files directly from these legacy archives without needing to unpack the entire archive to disk.

Here are the specific requirements:

1. **Configuration**: Your C++ server must read its configuration from `/etc/backup_config.json` on startup. The JSON will have the following structure:
   ```json
   {
       "archive_path": "/var/backups/production.dat",
       "index_path": "/var/backups/production.csv",
       "port": 9000
   }
   ```

2. **Index Parsing**: The `index_path` points to a CSV file generated by the legacy archiver. It contains metadata about the archived files (you will need to infer its exact column structure by running the `/app/legacy_archiver` binary on some test data).

3. **Data Decompression/Extraction**: The `archive_path` points to the binary blob. The legacy archiver applies a rudimentary proprietary "compression" (obfuscation) to the file bytes. You must reverse-engineer this simple algorithm by analyzing the input and output of `/app/legacy_archiver` and implement the corresponding decompression in C++.

4. **HTTP Interface**:
   - The service must listen on `127.0.0.1` on the port specified in the config (e.g., `9000`).
   - `GET /file/{filename}`: Should search the index for the given filename.
     - If found, seek to the appropriate location in the binary archive, read the compressed bytes, decompress them in memory, and return the plaintext file content with an HTTP 200 OK status.
     - If not found, return an HTTP 404 Not Found status.
   - You may use lightweight single-header C++ libraries (like `httplib.h` or `nlohmann/json.hpp`) by downloading them, or use standard POSIX sockets and minimal parsing.

5. **Execution**: Compile your code to `/usr/local/bin/archive_server`. Start the server in the background so that it is running and listening on the configured port.

*Note: Ensure your C++ code efficiently seeks into the binary file rather than reading the entire archive into memory, as production archives can be extremely large.*
