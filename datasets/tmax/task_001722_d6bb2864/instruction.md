You are an incident responder investigating an active exploitation campaign against your company's custom SSO service. Attackers are successfully bypassing your redirect validation filters to launch open-redirect and authentication-bypass attacks.

Your network forensics team intercepted a threat actor's communications and extracted an audio clip found at `/app/intercepted_call.wav`. We believe this audio reveals the compromised cryptographic salt the attackers are using to forge authentication tokens.

Your task is to build a standalone C-based detection utility to triage a corpus of captured HTTP requests.

Requirements:
1. Identify the compromised HMAC salt by analyzing `/app/intercepted_call.wav`.
2. Write a C program at `/home/user/detector.c` and compile it to `/home/user/detector` (you may link against OpenSSL using `-lcrypto`).
3. Your detector must take exactly one argument: the file path to a raw HTTP request.
   Example: `./detector /path/to/request.txt`
4. The detector must parse the HTTP request to extract the values of the `X-Redirect-URL` and `X-Auth-Token` headers. 
5. The `X-Auth-Token` is supposed to be an HMAC-SHA256 hash (in lowercase hex) of the `X-Redirect-URL` value, keyed with the compromised salt from the audio.
6. The detector must print exactly `CLEAN\n` to standard output and exit with code 0 if:
   - Both headers are present.
   - The token is a mathematically valid HMAC-SHA256 of the URL using the compromised salt (meaning it was generated by our system or the attackers).
   - AND the domain of the URL is strictly `trusted.corp` or a subdomain (e.g., `app.trusted.corp`). URLs using other domains (e.g., `evil.com`, `trusted.corp.evil.com`) are open redirect attempts.
7. The detector must print exactly `EVIL\n` to standard output and exit with code 1 if:
   - The headers are missing.
   - The HMAC token is invalid/incorrect.
   - The HMAC token is correct, but the URL points to a domain outside `trusted.corp` or its subdomains.

You can assume URLs in the headers will start with `https://` or `http://`.
We have provided sample corpora in `/app/corpus/clean/` and `/app/corpus/evil/` for you to test your detector. Your final solution must correctly classify 100% of both sets.
