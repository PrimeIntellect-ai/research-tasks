You are a data engineer building a lightweight ETL pipeline. We have a raw data feed from a sensor network stored in a text file at `/home/user/raw_sensor_data.txt`. The data contains unordered records of sensor readings with the format: `sensor_id timestamp value` (separated by spaces).

Your goal is to build an orchestration pipeline using a `Makefile` and a custom C program to sort the data, detect anomalies, and bulk load the results into an SQLite database.

Here are the exact requirements:

1. **C Program (`/home/user/process.c`)**:
   - Write a C program that reads `/home/user/raw_sensor_data.txt`.
   - The program must sort the data primarily by `sensor_id` (ascending) and secondarily by `timestamp` (ascending).
   - Detect "changepoint" anomalies: An anomaly occurs when a sensor's value differs from its *immediate chronological predecessor* (for the same `sensor_id`) by strictly more than `20.5` in absolute value.
   - Output two comma-separated files:
     - `/home/user/sorted_data.csv`: Contains all the sorted data in the format `sensor_id,timestamp,value` (print values to 2 decimal places).
     - `/home/user/anomalies.csv`: Contains only the anomalous records and the previous value in the format `sensor_id,timestamp,value,prev_value` (print values to 2 decimal places).

2. **Database Bulk Load**:
   - The pipeline needs to load the generated CSVs into a new SQLite database at `/home/user/sensor_db.sqlite`.
   - The database should have two tables:
     - `sensor_data` (sensor_id INTEGER, timestamp INTEGER, value REAL)
     - `anomalies` (sensor_id INTEGER, timestamp INTEGER, value REAL, prev_value REAL)
   - You must use the `sqlite3` CLI bulk import capabilities (e.g., `.import`) to populate these tables from the CSVs generated by your C program.

3. **Orchestration (`/home/user/Makefile`)**:
   - Create a `Makefile` that orchestrates this DAG. It must define the following targets:
     - `build`: Compiles `/home/user/process.c` into an executable named `processor`.
     - `process`: Runs `./processor` to generate the CSV files. (Depends on `build`).
     - `load`: Creates the SQLite database, creates the tables, and bulk imports the CSV files. (Depends on `process`).
     - `all`: Runs the entire pipeline (Depends on `load`).

When you are finished, run `make all` to execute the full pipeline. The final state must have the populated SQLite database at `/home/user/sensor_db.sqlite`.
