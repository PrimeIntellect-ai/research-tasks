You are a data scientist working on an experimental physics project. We have a hardware sensor that records particle collision events. Recently, a subtle electronic artifact ("ghosting") has started corrupting some of our data. 

We have isolated a proprietary, compiled simulator of the sensor from the hardware vendor, located at `/app/sensor_sim`. This is a stripped binary that acts as an oracle: when you run it (e.g., `/app/sensor_sim 42`), it generates a perfectly clean, baseline HDF5 file named `sim_output.h5` based on the provided integer seed. The data is stored in the HDF5 dataset named `/sensor_data`.

Your task is to write a Python CLI tool that acts as an anomaly detector to filter out corrupted experimental runs. 

1. Use Jupyter notebooks or Python scripts locally to visualize and compare the statistical distributions of the clean data (generated by the oracle) against potentially anomalous data.
2. Develop a density estimation or statistical hypothesis testing routine (e.g., using `scipy.stats`) to distinguish between the baseline physical distribution and the corrupted distribution. 
3. Write a final Python script at `/home/user/detector.py` that takes a single HDF5 file path as its argument.
4. The script must read the `/sensor_data` array from the given HDF5 file.
5. If the data is statistically "clean" (matching the oracle's distribution physics), the script must exit with status code `0`.
6. If the data contains the ghosting artifact (adversarial/corrupted), the script must exit with status code `1`.

We have provided a small sample of clean and corrupted files in `/app/training_data/clean/` and `/app/training_data/evil/` for you to explore and fit your models. 

Your script `/home/user/detector.py` will be tested against a hidden, held-out adversarial corpus of clean and evil HDF5 files to verify its accuracy and robustness.
