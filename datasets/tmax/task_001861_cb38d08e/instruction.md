You are a DevSecOps engineer tasked with enforcing Policy-as-Code and auditing legacy authentication mechanisms. 

Your team found a legacy authorization artifact in the form of Python bytecode disassembly, along with a sample token generated by this legacy system. The system generates authentication tokens that embed a user ID and a Content Security Policy (CSP) string. We suspect the token generator uses a hardcoded secret and a predictable hashing mechanism, and that the embedded CSPs violate strict security standards.

**Your objectives are:**

1. **Reverse Engineering:** Analyze the provided Python disassembly file located at `/home/user/app/token_signer.dis`. Deduce the token generation algorithm, the cryptographic hashing function used, and the hardcoded secret key.
2. **Token Validation & Policy Enforcement:** Write a Python script `/home/user/enforcer.py` that reads the sample token from `/home/user/app/sample_token.txt`. The script must:
   - Validate the token's cryptographic checksum using the recovered secret.
   - Decode the token to extract the user and the CSP.
   - Enforce a CSP security policy: Check if the token's CSP contains `unsafe-inline` or `unsafe-eval` within the `script-src` directive. If it does, the policy enforcement should fail.
3. **Token Generation:** Using the reverse-engineered logic and secret, forge a new valid token for the user `admin-pipeline` with a strictly compliant CSP: `default-src 'self'; script-src 'self';`.
4. **Reporting:** Your script must output the final results into a JSON file at `/home/user/security_report.json` exactly matching the following schema:
```json
{
  "recovered_secret": "the_plaintext_secret_found_in_disassembly",
  "sample_token_valid": true_or_false,
  "sample_token_policy_failed": true_or_false_if_it_contains_unsafe_directives,
  "forged_admin_token": "your_newly_generated_token_string"
}
```

**Environment specifics:**
- Everything should happen within `/home/user/`.
- The sample files are pre-staged in `/home/user/app/`.
- You may use any standard Python libraries. 
- Execute your script to generate the `/home/user/security_report.json` file.
