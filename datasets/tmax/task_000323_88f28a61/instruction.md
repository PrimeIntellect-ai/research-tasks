You are tasked with fixing a broken deployment of a custom C++ web backend behind an Nginx reverse proxy. Currently, making a request to the Nginx server on port 8000 returns a "502 Bad Gateway" error, or fails entirely, because the backend is misconfigured and constantly crashing.

The system uses a local Git deployment strategy. Here is the current state of the system:
1. Nginx is configured to run entirely in user-space using the configuration file located at `/home/user/nginx/nginx.conf`. It listens on port 8000.
2. A bare Git repository exists at `/home/user/repo.git`.
3. The C++ source code for the backend is checked into this repository.

Your objectives:
1. **Fix the Nginx Configuration**: The `nginx.conf` file has a typo in its proxy configuration. Find it and fix it so it points to the C++ backend, which should be listening on port 8080.
2. **Directory Management**: The C++ backend expects to write logs to a `logs/` directory inside its working directory. Create a directory at `/home/user/app_logs` and ensure that when the application runs, it uses a symlink at `/home/user/app/logs` pointing to `/home/user/app_logs`.
3. **Git Hook & Automation**: Create a Git `post-receive` hook at `/home/user/repo.git/hooks/post-receive`. This hook must be an idempotent bash script that does the following upon receiving a push:
   - Checks out the latest code into `/home/user/app`.
   - Compiles the C++ backend (`server.cpp`) into an executable named `backend`.
   - Gracefully stops the old backend process if it's running.
   - Starts the new backend process in the background, listening on port 8080.
   - **Crucially**, the backend relies on executing a system command (`date`) to log its startup. However, the existing startup mechanism suffers from a PATH issue. Ensure the hook sets a fully qualified `PATH` and sets the `TZ` environment variable to `UTC` before starting the C++ process.
4. **Fix the Backend and Deploy**: Clone the repository from `/home/user/repo.git` to a temporary workspace, fix any pathing or environment bugs in the C++ code if necessary, commit your changes, and push them to trigger your new `post-receive` hook.
5. **Start Nginx**: Finally, start Nginx using the fixed configuration: `nginx -c /home/user/nginx/nginx.conf`

Verification:
When you are finished, a `curl -s http://127.0.0.1:8000` must successfully return an HTTP response generated by the C++ backend, the background process must be stable, and `/home/user/app_logs/startup.log` must contain a UTC timestamp.
