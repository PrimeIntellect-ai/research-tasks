You are a network engineer analyzing traffic from a proprietary IoT device on your network. You suspect the device is using a vulnerable, homegrown challenge-response authentication protocol.

You have been provided with the following files in your home directory (`/home/user/`):
1. `traffic_capture.log` - A text-based extract of network packets showing recent authentication attempts. 
2. `auth_algo.py` - A Python script recovered from the device's firmware that contains the `generate_response(password, challenge)` function used by the authentication protocol.
3. `wordlist.txt` - A dictionary file containing a list of potential passwords.
4. `next_challenge.txt` - A file containing the server's next challenge string for the `admin` user, which you just intercepted.

Your task is to:
1. Parse `traffic_capture.log` to find a successful authentication attempt for the `admin` user. Extract the challenge and the response.
2. Using the `auth_algo.py` logic and the provided `wordlist.txt`, write a Python script to brute-force the `admin` password by recreating the hashing process until the output matches the intercepted response.
3. Once you have recovered the `admin` password, use it to calculate the correct authentication response for the challenge found in `next_challenge.txt`.
4. Save *only* the final computed response string (in lowercase hex, as generated by the algorithm) to a file named `/home/user/admin_response.txt`.

Ensure your final file `/home/user/admin_response.txt` contains no extra whitespace, newlines, or text other than the exact response string.
