You are a data scientist dealing with a messy dataset generated by an unstable ETL pipeline. The ETL job frequently failed and retried, creating duplicate records. However, because of how the data was ingested, the "duplicate" records aren't exactly identical—they have slight variations in their text fields.

Your goal is to create an automated pipeline that deduplicates this dataset and generates a summary report.

**Task Requirements:**

1. **Input Data:** 
   You will process a file located at `/home/user/etl_data.tsv`. It is a tab-separated values (TSV) file with the following columns (no header row):
   `[ID]    [Category]    [Timestamp (Epoch)]    [Message]`

2. **Deduplication Logic:**
   Create a Bash script at `/home/user/clean_pipeline.sh` that processes this file. You may use inline Python, Perl, or Awk within or alongside your Bash script.
   * **Group and Sort:** First, group the records by `Category`. Within each category, sort the records by `Timestamp` in ascending order.
   * **Similarity Computation:** Iterate through the sorted records in each category. A record is considered a "retry duplicate" (and should be dropped) if its `Message` has a Levenshtein distance of **3 or less** compared to the `Message` of any *already kept* record in the same `Category`.
   * **Output:** Save the kept records (in their sorted order, preserving the original TSV format) to `/home/user/cleaned_data.tsv`.

3. **Template-Based Reporting:**
   Your script `/home/user/clean_pipeline.sh` must also generate a summary report at `/home/user/report.txt` exactly matching this template:
   ```
   === ETL Deduplication Report ===
   Total Input Records: <N>
   Total Cleaned Records: <M>
   Total Duplicates Dropped: <K>
   ================================
   ```
   (Replace `<N>`, `<M>`, and `<K>` with the actual integer counts).

4. **Pipeline Scheduling:**
   Finally, schedule your script to run automatically by adding it to the user's crontab. It must be scheduled to run every day at exactly **3:15 AM**.

Make sure your script `/home/user/clean_pipeline.sh` is executable and runs successfully on the provided data, generating the correct output files before you finish.
