You are a data analyst working for a global e-commerce company. You have received a batch of customer review data from different regional offices. The data consists of several CSV files located in `/home/user/data/reviews/`. 

Because these files were generated by different regional systems, they use different character encodings (e.g., UTF-8, Latin-1, Shift-JIS, Windows-1251). Your task is to extract mentions of product codes from the text of these reviews and count their frequencies.

A valid product code is defined by the regular expression `\b[A-Z]{2,3}-\d{4,5}\b` (e.g., `AB-12345`, `XYZ-9876`).

Please write a Python script at `/home/user/process_reviews.py` that performs the following:
1. Iterates over all CSV files in `/home/user/data/reviews/`.
2. Reads each CSV file. You must implement robust character encoding handling to read the files without throwing `UnicodeDecodeError` (you may use libraries like `chardet`, or try a fallback sequence of common encodings). The CSVs have a header row and two columns: `review_id` and `review_text`.
3. Processes the files in parallel using Python's `multiprocessing` or `concurrent.futures` modules to speed up the extraction.
4. Extracts all product codes from the `review_text` column across all files.
5. Aggregates the total count of each unique product code.
6. Saves the final aggregated counts to a JSON file at `/home/user/product_frequencies.json`. The JSON should be a dictionary where the keys are the product codes and the values are their integer counts.

Once you have written the script, run it so that the `/home/user/product_frequencies.json` file is generated. Ensure that any necessary third-party packages (like `chardet` if you choose to use it) are installed via pip.
