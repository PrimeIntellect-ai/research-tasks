You are a data engineer working on an ETL pipeline that processes multi-dimensional sensor data. A crucial step in this pipeline is a dimensionality reduction transformation, which requires calculating the principal components of the data using linear algebra. 

Because our backend is written in Go, you need to implement this transformation step in Go using the `gonum` numerical library. Furthermore, to ensure there are no floating-point discrepancies or library configuration issues, you must validate the numerical accuracy of your results against a set of reference values generated by our legacy Python system.

**Your Tasks:**

1. **Environment Setup:**
   - Create a Go module named `etl_transform` in the `/home/user/etl_transform` directory.
   - Install the `gonum.org/v1/gonum` library.

2. **Data Ingestion & Linear Algebra Transformation:**
   - Write a Go program (`/home/user/etl_transform/main.go`) that reads `/home/user/sensor_data.csv`. This file contains 100 rows and 5 columns of floating-point sensor readings (no header row).
   - Using `gonum`, calculate the 5x5 covariance matrix of this dataset. Treat each column as a variable and each row as an observation. *Hint: You may use `stat.CovarianceMatrix` or compute it manually after mean-centering.*
   - Compute the eigenvalues of this covariance matrix. Since it's a symmetric matrix, all eigenvalues will be real numbers. You should use `mat.EigenSym`.
   - Sort these eigenvalues in **descending** order.

3. **Numerical Accuracy Testing:**
   - Read the reference eigenvalues from `/home/user/reference_eigenvalues.txt` (which contains 5 space-separated floating-point numbers, already sorted in descending order).
   - Compare your computed eigenvalues to the reference values. The numerical accuracy test passes if the absolute difference for *every* corresponding eigenvalue is strictly less than or equal to `1e-4`.

4. **Output Generation:**
   - Your Go program must output a JSON file at `/home/user/etl_transform/report.json` with exactly this structure:
     ```json
     {
       "eigenvalues": [val1, val2, val3, val4, val5],
       "accuracy_passed": true
     }
     ```
     (Where `val1` to `val5` are your computed eigenvalues in descending order, and `accuracy_passed` is a boolean representing the result of your accuracy test).

Write, build, and run your Go program to generate the required `report.json` file.
