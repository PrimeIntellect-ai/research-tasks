You are tasked with building a configuration manager that tracks changes, minifies configuration data, and securely archives it. We have a multi-service setup located in `/app/` consisting of an Nginx server (serving base templates on port 8080) and a Flask webhook receiver (running on port 5000). 

Your objective is to write a Python script at `/home/user/config_tracker.py` that fulfills the following requirements:

1. **Watch and Detect:** The script must monitor the directory `/home/user/config_mount/` for any new or modified `.json` files. This directory simulates a configuration mount.
2. **Parse and Optimize:** Whenever a JSON file is created or modified, your script must parse the JSON. You must remove all whitespace, newlines, and null values to minimize the payload size.
3. **Archive:** The optimized JSON must be archived into a single gzip-compressed tarball named `/home/user/archives/config_history.tar.gz`. Each updated file should be appended or updated in this archive.
4. **Notify:** After updating the archive, the script must send a POST request to `http://127.0.0.1:5000/webhook` with the JSON payload: `{"latest_update": "<filename>", "archive_size": <size_in_bytes>}`.

**Constraints & Verification:**
- You must create the `/home/user/config_mount/` and `/home/user/archives/` directories.
- To pass verification, your script will be tested against a barrage of 50 complex, highly-nested JSON configuration files generated by our test suite. 
- **Metric Requirement:** The final `/home/user/archives/config_history.tar.gz` file must have a total size of strictly less than `15000 bytes` after all 50 files are processed. This requires aggressive JSON minification (removing all whitespace, indentation, and null keys) before archiving. 
- You must leave your script running in the background or provide instructions in a `/home/user/run.sh` script so our test suite can start it, emit the files, and verify the webhook and archive size.
