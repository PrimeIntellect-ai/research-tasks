You are an algorithmic bioinformatics analyst tasked with modeling a cyclic biological process. We have extracted feature matrices representing environmental drivers, but due to redundant sequencing data, the design matrix is near-singular, causing standard regression to fail. 

Your objective is to estimate the biological parameters, simulate the system dynamics via an ODE solver to test convergence, and perform spectral analysis on the resulting sequence.

**Phase 1: Parameter Estimation (Near-Singular Matrix)**
You are provided with two files:
- `/home/user/design_matrix.csv` (100 rows, 3 columns)
- `/home/user/target.csv` (100 rows, 1 column)

The linear model is $X p = y$. Because the columns of $X$ are highly collinear, standard OLS will fail or overfit. 
1. Use Python to compute the Moore-Penrose pseudoinverse of $X$ with a relative condition number cutoff (`rcond`) of exactly `1e-2`. 
2. Multiply this pseudoinverse by $y$ to estimate the parameter vector $p = [p_0, p_1, p_2]$. 

**Phase 2: ODE Numerical Solving & Convergence Testing**
The biological process is governed by the following Ordinary Differential Equation (ODE):
$$ \frac{dz}{dt} = -p_0 z + p_1 \cos(p_2 t) $$
with initial condition $z(0) = 1.0$.

1. Implement the **Forward Euler method** to numerically solve this ODE from $t = 0$ to $t = 100$.
2. To test for numerical convergence, run the solver using three different time steps: $dt = 1.0$, $dt = 0.1$, and $dt = 0.01$.
3. Record the final value of the simulation $z(100)$ for each time step.

**Phase 3: Spectral Analysis**
1. Using the trajectory generated by the $dt = 0.01$ simulation (which contains 10,001 points including $t=0$), perform a Fast Fourier Transform (FFT) on the $z(t)$ series.
2. Calculate the corresponding frequencies in Hertz (cycles per unit time).
3. Identify the dominant frequency $f_{peak}$ (the frequency $> 0$ with the maximum spectral magnitude). 

**Requirements:**
Write a main Bash script `/home/user/analyze.sh` that orchestrates this entire pipeline (it may call embedded or standalone Python scripts). The script must create a final report file at `/home/user/results.txt` with exactly 5 lines:
Line 1: The estimated parameters comma-separated: `p_0, p_1, p_2` (rounded to 4 decimal places)
Line 2: $z(100)$ for $dt=1.0$ (rounded to 4 decimal places)
Line 3: $z(100)$ for $dt=0.1$ (rounded to 4 decimal places)
Line 4: $z(100)$ for $dt=0.01$ (rounded to 4 decimal places)
Line 5: The dominant frequency $f_{peak}$ in Hz (rounded to 4 decimal places)

Ensure your script is executable and performs all calculations when run.
