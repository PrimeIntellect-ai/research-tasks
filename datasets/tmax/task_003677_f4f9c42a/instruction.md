You are an MLOps engineer tasked with fixing our internal experiment tracking tool and writing a data sanitization pipeline for our large-scale experiment logs.

We have a custom Python package for artifact tracking located at `/app/ml_artifact_tracker-0.1.0`. Recently, we migrated to a new Python version, and the package currently fails to install or run properly due to an import error and a strict, broken dependency in its `setup.py`. 
Your first step is to fix the source code of this package and install it in your environment using `pip install -e /app/ml_artifact_tracker-0.1.0`.

Second, we have a large volume of experiment logs generated by multiple legacy systems. Many of these logs are corrupted or represent failed, incomplete experiments. You must build a script that filters out the corrupted logs.

Write a script named `/home/user/detect_corrupted.py` that can be invoked exactly like this:
`python /home/user/detect_corrupted.py --input-dir <path_to_directory> --output <path_to_output_csv>`

The script must read all JSON files in `<path_to_directory>`, extract their metadata, and identify the "clean" experiments. 
A "clean" (valid) experiment artifact strictly satisfies ALL of the following conditions:
1. `schema_version` is exactly `"v2"`.
2. `status` is exactly `"SUCCESS"`.
3. `metrics.loss` exists, is a valid number (not NaN), and is `<= 2.0`.
4. `metrics.accuracy` exists, is a valid number, and is `>= 0.5`.
5. `model_size_mb` (which might be nested under `artifacts` or at the root level, you must check both) exists, is an integer (or convertible to one without data loss), and is `>= 10`.

Any JSON file that fails one or more of these criteria, is missing required fields, or has malformed JSON is considered "corrupted".

The script must write the `artifact_id` (a string field present in all JSONs) of ONLY the clean artifacts to the file specified by `--output`. The output file should be a simple text file with one `artifact_id` per line, sorted alphabetically.

For testing during your development, you can create your own dummy JSON files. When your solution is evaluated, we will run your script against our hidden evaluation corpora of clean and corrupted JSON logs.
