You are helping a technical writing team organize legacy documentation. The documentation is continuously generated by an old system that writes to a custom "Rotated Archive Stream" (RAS) format. Because the logging system races with a rotation process, the stream often contains random garbage bytes between valid document chunks.

Your task is to write a Python script at `/home/user/doc_extractor.py` that reads a raw binary stream from `stdin` and extracts the valid compressed documents to `stdout`.

We have a screenshot of the legacy system's original specifications in an image located at `/app/legacy_spec.png`. You will need to extract the 4-byte "Magic Header" value from this image, as it is required to parse the binary stream.

### RAS Binary Format Specification
The binary stream may contain garbage at any point. Your script must scan through the stream byte-by-byte to find the 4-byte Magic Header (found in the image).
Once the 4-byte magic header is found, the immediately following bytes represent a potential chunk:
1. **Length (`N`)**: A 4-byte unsigned little-endian integer representing the length of the compressed payload.
2. **Payload**: `N` bytes of zlib-compressed data.
3. **Checksum**: A 4-byte unsigned little-endian integer representing the CRC32 checksum of the **uncompressed** payload.

### Extraction and Verification Logic
When you encounter a potential magic header:
1. Read the length `N`.
2. Read the `N` bytes of payload. If there aren't enough bytes left in the stream, this chunk is invalid.
3. Read the 4-byte CRC32.
4. Attempt to zlib-decompress the payload. If decompression fails, the chunk is invalid.
5. Compute the CRC32 of the uncompressed data. If it does not match the checksum in the chunk, the chunk is invalid.

If a chunk is valid, process it as described below. 
If a chunk is invalid at any stage (unexpected EOF, zlib error, or CRC32 mismatch), you must discard it and resume scanning for the magic header starting from the byte **immediately following** the magic header that initiated the failed attempt.

### Processing Valid Documents
The uncompressed data is a UTF-8 string. The first line of every valid document will always be of the format: `Title: <DocumentName>`. The rest of the document follows on subsequent lines.
For every valid chunk, you must extract the `<DocumentName>`, and print to `stdout` exactly in this format:
```
=== <DocumentName>.txt ===
<The rest of the document's content (excluding the Title line)>
---DOC---
```
Ensure there is exactly one newline character after `---DOC---`.

Write your executable script to `/home/user/doc_extractor.py`. It should read the entire binary stream from `stdin` until EOF, and write only the formatted valid documents to `stdout`.
