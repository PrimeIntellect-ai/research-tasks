You are a data scientist tasked with cleaning up a dataset generated by a faulty ETL pipeline. The pipeline processes system state changes but occasionally retries upon failure, resulting in exact duplicate rows. Furthermore, the downstream analytics team requires the data on a strict hourly grid, but the raw events are irregularly spaced.

Your task is to write a **C program** that reads the raw event log, parses the timestamps, removes duplicates, and resamples the data to a regular hourly grid using forward-filling.

**Input Data:**
A CSV file located at `/home/user/raw_events.csv`. 
Format: `YYYY-MM-DD HH:MM:SS,STATE`
*   The file is chronologically sorted.
*   The `STATE` is an uppercase string (e.g., `ACTIVE`, `WARNING`, `ERROR`, `MAINTENANCE`).
*   Some adjacent rows are exact duplicates due to ETL retries.

**Processing Requirements:**
1.  **Timestamp Parsing & Alignment:** Parse the timestamps. Generate a strict hourly grid starting exactly at `2023-10-01 00:00:00` and ending at `2023-10-01 23:00:00` (inclusive, 24 rows total).
2.  **Resampling & Gap-Filling:** For each hour on the grid, output the system's state exactly at that hour. 
    *   The state at a given hour is the state of the *most recent* event that occurred at or before that hour.
    *   If no events have occurred prior to a given hour on the grid, the state defaults to `OFFLINE`.
3.  **Duplicate Handling:** Ignore redundant state events (ETL retry duplicates).

**Output:**
Write the results to `/home/user/hourly_status.csv` with the exact format:
`YYYY-MM-DD HH:MM:SS,STATE`

**Implementation Constraints:**
*   You must implement the logic in C.
*   Save your source code as `/home/user/process.c`.
*   Compile it to an executable named `/home/user/process` and run it to produce the output file.
