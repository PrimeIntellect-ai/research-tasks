You are an infrastructure engineer stepping into a crisis. Our legacy configuration management system crashed during a backup process due to an infinite symlink loop. We lost the master configuration database, but we have a directory of raw backup archives and a video recording of the terminal when the crash occurred.

Your task is to recover the valid configurations, normalize them, and produce a unified recovery report.

Here is what you need to do:

1. **Video Analysis:**
   The terminal screencast of the crash is located at `/app/symlink_crash.mp4`. Use `ffmpeg` and Python (e.g., OCR or frame analysis) to find the exact frame where the fatal error occurs. The error message will explicitly state the archive ID that caused the infinite symlink loop (format: `backup_[ID].tar.gz`). Note this ID.

2. **Archive Integrity Verification:**
   Inside `/app/backups/`, there are numerous `.tar.gz` configuration backup files. Many of them were corrupted during the network partition that followed the crash. 
   Write a Python script to verify the integrity of all `.tar.gz` files in this directory. 
   **Discard** any archives that are corrupted AND **discard** the specific archive ID you identified from the video (as it contains the symlink loop payload).

3. **Data Recovery and Encoding Conversion:**
   For all valid archives, extract the single configuration file inside each. These files were generated by legacy systems across different regions, so their filenames and contents are in various encodings (specifically `Shift_JIS` and `cp1252`). 
   Using streaming or memory-mapped I/O (as some of these files are padded with large amounts of junk data), read the files, convert their contents to valid UTF-8, and extract only the lines that begin with `CONFIG_SET: `.

4. **Bulk File Renaming:**
   Save the extracted, UTF-8 encoded configuration lines into a new directory: `/app/recovered_configs/`.
   You must bulk rename these files following this exact format: `config_<archive_id>.txt` (e.g., `config_1024.txt`), regardless of their original garbled filenames inside the archives.

5. **Reporting:**
   Compile the recovered configuration parameters into a single JSON file at `/app/recovery_report.json`. 
   The format must be a dictionary where the keys are the archive IDs (as strings) and the values are a list of the extracted `CONFIG_SET: ` line contents (stripped of the "CONFIG_SET: " prefix and trailing newlines).

Your final output will be evaluated automatically by comparing the contents of `/app/recovery_report.json` against a hidden ground-truth file. A strict string-matching metric will be used to calculate your accuracy. You must achieve an accuracy score of at least 0.95 (95%).
