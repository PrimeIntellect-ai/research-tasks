Due to unstable floating-point reduction order in its numerical ODE solver, the simulated energy sometimes diverges, causing the trajectory's amplitude to grow over time instead of decaying as expected for a damped system. 

I need an automated way to sanitize this training data. I have provided a sample of valid (convergent) trajectories in `/home/user/data/clean/` and a sample of invalid (divergent) trajectories in `/home/user/data/evil/`. Every file is a CSV with headers `time,x,v`.

Your task is to write a Bash script at `/home/user/filter.sh` that determines whether a given simulation output is valid. 
- The script must take exactly one argument: the path to a CSV file.
- It must exit with status code `0` if the file represents a clean, stable trajectory.
- It must exit with status code `1` if the file represents an evil, unstable trajectory.
- You must write the logic strictly in Bash, utilizing standard Linux CLI tools (like `awk`, `sed`, `grep`, `bc`, etc.). Do not use Python, Perl, or other higher-level scripting languages for the final `/home/user/filter.sh` script.

You can run the stripped binary `/app/sim_generator` if you need to reverse-engineer its behavior or generate more test samples.

The automated verification system will test your script against a hidden holdout set of both clean and evil CSVs. To pass, your script must successfully preserve (exit 0) 100% of the hidden clean corpus and reject (exit 1) 100% of the hidden evil corpus.
