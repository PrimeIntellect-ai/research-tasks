Your goal is to build a Python pipeline that standardizes this data, analyzes the text logs, and generates a summary report.

1. **Wide-to-Long Reshaping:**

   Create a new column called `normalized_log`. For each string in the `log` column:
   - Convert to lowercase.
   - Remove all punctuation (keep only alphanumeric characters and spaces).
   - Tokenize by splitting on spaces into a list of words. (e.g., `"Error: Valve 2!"` becomes `["error", "valve", "2"]`).

   Find all pairs of *distinct* non-empty logs in the newly resampled dataset that are highly similar. 
   - Use **Jaccard similarity** on the sets of tokens from `normalized_log`.
   - Jaccard similarity is (Intersection size) / (Union size).
   - Identify pairs where the Jaccard similarity is `>= 0.6` but `< 1.0` (not exact matches).
   - Record the original `log` text of these pairs. To avoid duplicates, always order the pair alphabetically by the original log string.

5. **Template-based Generation:**
   Save the cleaned dataframe to `/home/user/cleaned_data.csv`.

   ```markdown

   Average Temperature for Sensor B: {avg_temp_B}
   
   ## Similar Log Pairs

   ```
   *Note: Format floating-point numbers to 2 decimal places (e.g., 23.45). List similar log pairs sorted alphabetically by the first log.*
