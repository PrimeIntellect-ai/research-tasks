I am a researcher organizing a large batch of sensor dataset archives. Some of our dataset files have been corrupted by bad transfers, and some were generated by a faulty sensor node that injects unauthorized data or invalid manifests. I need you to create a dataset validation script to filter out the bad archives.

First, you will find an image of my lab notes at `/app/lab_notes.png`. You need to read this image (you can use Tesseract OCR, which is installed) to extract the "Project Salt" string written on it.

Then, write a Python script at `/home/user/verifier.py` that takes a single command-line argument: the path to a directory containing zip archives.
For each `.zip` file in that directory, your script must determine whether to `ACCEPT` or `REJECT` it based on the following criteria:

1. **Archive Integrity:** The zip file must be a valid, uncorrupted zip archive.
2. **Manifest Presence & Completeness:** The zip must contain a file named `manifest.txt`. Every other file in the zip (except `manifest.txt` and `signature.txt`) must be listed in this manifest. There must be no extra files in the zip that aren't in the manifest, and no files listed in the manifest that are missing from the zip.
3. **Checksum Verification:** The `manifest.txt` contains lines formatted as `<filename> <sha256_hex_digest>`. The calculated SHA256 checksum of each file must perfectly match the one listed in the manifest.
4. **Signature Validation:** The zip must contain a `signature.txt` file. This file must contain exactly the SHA256 hex digest of the raw bytes of `manifest.txt` immediately followed by the "Project Salt" string you extracted from the lab notes. (e.g., if the salt was "SALT", the signature is `sha256(manifest_bytes + b"SALT")`).

Your script must iterate through all `.zip` files in the provided directory in alphabetical order. For each file, print exactly one line to standard output in this format:
`[filename]: ACCEPT` or `[filename]: REJECT`

To help you develop and test your script, I have provided a directory of known good datasets at `/app/data/clean/` and a directory of known bad datasets at `/app/data/evil/`. 
Your script must correctly `ACCEPT` 100% of the archives in the `clean` directory and `REJECT` 100% of the archives in the `evil` directory. 

Once your script is complete, test it thoroughly. An automated verification suite will later invoke your script against entirely new, hidden clean and evil corpora to ensure it properly implements all rules.
