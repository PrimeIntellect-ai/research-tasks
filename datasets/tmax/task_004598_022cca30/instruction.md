I am migrating a legacy data processing pipeline from Python 2 to Python 3. As part of this migration, we decided to rewrite the log auditing module—which validates past requests and checks our rate limiting rules—in C for better performance.

I have a binary log file located at `/home/user/requests.bin` generated by the old Python 2 system. I need you to write a C program that deserializes this binary file, applies our rate-limiting algorithm to validate the requests, and outputs a text report. I also need you to set up a basic CI pipeline via a `Makefile`.

Here are the detailed requirements:

**1. Deserialization:**
The `/home/user/requests.bin` file contains a sequence of records. Each record is exactly 10 bytes, packed in little-endian format:
- `timestamp`: 32-bit unsigned integer (seconds since epoch)
- `ip_address`: 32-bit unsigned integer (IPv4 address as an integer)
- `endpoint_id`: 16-bit unsigned integer

**2. Request Validation & Rate Limiting (Algorithm):**
Your C program must implement a strict "Fixed Window" rate limiter algorithm to audit the logs.
- The fixed window size is exactly 1 second (based on the `timestamp`).
- A single `ip_address` is allowed a maximum of **3 requests** per second (across all endpoints).
- Process the records sequentially. For each record, if the IP address has already made 3 accepted requests in that exact 1-second timestamp window, the request is `REJECTED`. Otherwise, it is `ACCEPTED`.
- The state resets completely at the start of a new timestamp (i.e., when the timestamp changes, the count for all IPs resets to 0). Note: The records in the binary file are strictly ordered by timestamp chronologically.

**3. Output Format:**
The C program must be saved as `/home/user/audit.c`. It should write its output to `/home/user/audit_results.txt`.
For each record in the binary file, write exactly one line with the format:
`<timestamp> <ip_address> <STATUS>`
Where `<STATUS>` is either `ACCEPTED` or `REJECTED`.

*Example output line:*
`1609459200 3232235521 ACCEPTED`

**4. CI/CD Pipeline Setup:**
Create a `/home/user/Makefile` with the following targets:
- `all`: Compiles `audit.c` into an executable named `audit` using `gcc -O2 -Wall`.
- `clean`: Removes the `audit` executable and `audit_results.txt`.
- `test`: Depends on `all`. It should run `./audit`, then use `diff -q audit_results.txt expected_results.txt`. If the diff succeeds, it should print "CI Passed".

(Assume `/home/user/expected_results.txt` already exists in the environment for the test).

Please create the C file and the Makefile, and ensure `make test` runs successfully.
