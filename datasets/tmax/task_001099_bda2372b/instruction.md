You are an operations engineer triaging a critical incident. A legacy data validation system has stopped working after a recent server migration, and the original developer is unreachable. 

The system relies on a compiled binary located at `/home/user/bin/validator`, which currently fails and outputs "ERROR: Validation failed." 

Through preliminary investigation, you've discovered that the binary requires specific environment variables to function correctly:
1.  **A database password:** This password was previously version-controlled in a Git repository located at `/home/user/legacy_repo`. It was accidentally committed in a file called `db_config.txt`, but a subsequent commit deleted the file to "hide" the secret. You need to perform forensics on this Git repository to recover the original database password.
2.  **An authentication token:** The `validator` binary expects a hardcoded authentication token provided via an environment variable. You must reverse-engineer the `/home/user/bin/validator` binary (using tools like `strings`, `ltrace`, or similar) to determine the exact environment variable name it checks for and the string value it expects.

Your task is to:
1. Recover the database password from the git history.
2. Determine the expected authentication environment variable and its value from the binary.
3. Write a Bash script at `/home/user/triage.sh` that:
   - Exports the recovered database password as the `DB_PASS` environment variable.
   - Exports the recovered authentication token to the correct environment variable name expected by the binary.
   - Executes `/home/user/bin/validator`.
4. Run your script and redirect its successful standard output to `/home/user/incident_resolution.log`.

Your objective is complete when `/home/user/incident_resolution.log` contains the success message and flag generated by the binary. Ensure your `triage.sh` script is executable.
