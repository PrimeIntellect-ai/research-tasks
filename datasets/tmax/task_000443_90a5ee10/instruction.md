You are a platform engineer maintaining a CI/CD pipeline. One of the pipeline steps relies on a small C utility called `rpn_calc` to evaluate mathematical expressions (Reverse Polish Notation) generated by a metric collector. 

Currently, the utility is broken in two ways:
1. The project has a linking error and won't compile.
2. Even if forced to compile, the utility suffers from a memory safety issue (undefined behavior/buffer overflow) that causes it to crash with a segmentation fault when evaluating certain long expressions.

Your task is to fix the utility and process a dataset of expressions.

Setup:
The files are located in `/home/user/rpn_calc/`:
- `Makefile`: The build configuration.
- `rpn_calc.c`: The source code for the RPN calculator. It takes a single string argument representing the RPN expression and prints the evaluated result as a double.
- `inputs.txt`: A file containing one RPN expression per line.

Requirements:
1. Fix the `Makefile` so that it successfully compiles and links the `rpn_calc` executable. 
2. Fix the memory safety bug in `rpn_calc.c`. The internal stack size is fixed, but it currently lacks bounds checking. Modify the code so that if a stack overflow or underflow would occur, the program gracefully prints "Error" to `stderr` and exits with code `1`.
3. Build the `rpn_calc` binary.
4. Process `/home/user/rpn_calc/inputs.txt`. Evaluate each line using the fixed `rpn_calc` binary.
5. Discard any expressions that result in an Error (exit code 1).
6. Collect the successfully evaluated numerical results, sort them in strictly **descending** order (largest to smallest).
7. Save the sorted results to `/home/user/rpn_calc/sorted_outputs.txt`. Each number must be formatted to exactly 2 decimal places (e.g., `42.00`).

Do not change the underlying math logic of valid expressions, only the build configuration, the memory safety boundaries, and the pipeline processing script.
