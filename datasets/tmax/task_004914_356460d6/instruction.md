You are an integration developer tasked with replacing a legacy error-correction and validation microservice. 

Currently, our system relies on a Node.js producer service that generates telemetry payloads. Unfortunately, the network layer introduces bit-flips and corruption. The producer encodes the data using a custom Error-Correcting Code (ECC) and checksum algorithm written in JavaScript.

Your objective is to translate this legacy JavaScript ECC logic into Python, build a Python integration service that interfaces with the producer and a local cache, and rigorously test it.

**Environment & Setup:**
In `/app/fixture/`, you will find a multi-service setup script: `start_services.sh`. This script brings up:
1. **Telemetry Producer (Port 8081):** A Node.js-based HTTP server. When you `GET /telemetry`, it returns a JSON object containing `{ "id": "<uuid>", "raw_payload": "<hex_string>", "ecc_block": "<hex_string>" }`. The payload often contains injected errors.
2. **Key-Value Cache (Port 8083):** A lightweight Python-based cache holding validation keys.

The legacy JavaScript implementation of the ECC encoder/decoder is located at `/app/legacy/ecc.js`.

**Your Tasks:**
1. **Code Translation:** Translate the custom ECC logic from `/app/legacy/ecc.js` into a robust Python module located exactly at `/home/user/workspace/ecc.py`.
2. **Integration Service:** Create a Python API or script at `/home/user/workspace/integration.py`. This script must:
    - Query the Producer (`http://127.0.0.1:8081/telemetry`) 500 times.
    - Use your `ecc.py` module to detect and correct bit-flips in the `raw_payload` using the `ecc_block`.
    - Query the Cache (`http://127.0.0.1:8083/verify?id=<uuid>&corrected_payload=<hex>`) to verify the corrected payload.
    - Log the results of each operation to `/home/user/workspace/results.json` in the format: `[{"id": "...", "status": "success|failed"}, ...]`.
3. **Unit Testing:** Write a comprehensive unit test suite in `/home/user/workspace/test_ecc.py` that tests your `ecc.py` module. Ensure you cover edge cases, multiple bit flips (as defined in the JS limits), and checksum failures.

**Constraints & Verification:**
- You must use Python for the implementation and testing. You can use standard libraries and `pytest`/`requests` (install via pip if needed).
- An automated verifier will evaluate your implementation. It will evaluate the **metric_threshold** of your Error Recovery Rate. Your implementation must successfully correct and verify at least 95% of the payloads generated by a held-out evaluation producer.
- Ensure your background services are running properly by inspecting their logs if necessary.
