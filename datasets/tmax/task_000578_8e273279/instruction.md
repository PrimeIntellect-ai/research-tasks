You are assisting a technical writer who is organizing scattered API documentation notes. 

In the directory `/home/user/tech_logs/`, there is a nested folder structure (`year/month/`) containing multiple `.log` files. These files contain multi-line documentation drafts generated by an automated scraping tool that occasionally splits records across files during log rotation. 

The format of a log record looks exactly like this:
```
=== [DOC-ID] ===
Type: [Record Type]
Content:
[Multi-line text content]
[can be multiple lines]
================
```
Example:
```
=== DOC-4092 ===
Type: API_DEPRECATION
Content:
The /v1/users endpoint will be removed.
Migrate to /v2/users immediately.
================
```

Additionally, there is a JSON metadata file located at `/home/user/authors.json`. This file contains a dictionary mapping the `DOC-ID` to an object containing author information:
```json
{
  "DOC-4092": {"author": "j.smith@example.com", "tags": ["v1", "users"]},
  ...
}
```

Your task is to write and execute a Python script that does the following:
1. Navigates the `/home/user/tech_logs/` directory tree to read all `.log` files.
2. Parses the multi-line log records to find EVERY record where the `Type` is exactly `API_DEPRECATION`.
3. Extracts the corresponding `DOC-ID` for those records.
4. Looks up the author's email for each extracted `DOC-ID` using `/home/user/authors.json`.
5. Writes the aggregated results to a standard CSV file at `/home/user/deprecation_report.csv`. 

The output CSV must have exactly two columns with the headers: `DocID,Author`.
The rows in the CSV must be sorted alphabetically by the `DocID`.

Ensure the final CSV is correctly formatted, as an automated test will parse `/home/user/deprecation_report.csv` to verify your work.
