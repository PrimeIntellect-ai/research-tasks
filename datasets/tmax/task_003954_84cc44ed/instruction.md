You are an administrator tasked with integrating a modern Git workflow with a legacy backend system. You need to set up a Git repository that automatically syncs commit metadata to an interactive legacy TCP service via a custom pipeline.

Your objectives:

1. **Protocol Converter (`/home/user/protocol_converter.py`)**:
   Write a Python 3 script that reads text from `stdin` until EOF, parses it, and outputs commands to `stdout`. The script must act exactly according to these rules:
   - If a line starts with `COMMIT:` (e.g. `COMMIT:a1b2c3`), output `TRANSACTION START <hash>` (e.g. `TRANSACTION START a1b2c3`).
   - If a line starts with `AUTHOR:` (e.g. `AUTHOR:Jane Doe`), output `SET_USER <name>`.
   - If a line matches the regex `^([AMD])\s+(.+)$` (which corresponds to Git's `name-status` output), output:
     - For `A`: `FILE_ADD <filename>`
     - For `M`: `FILE_MOD <filename>`
     - For `D`: `FILE_DEL <filename>`
   - Any other line (including blank lines) must be completely ignored.
   - Example Input:
     ```
     COMMIT:abc1234
     AUTHOR:Sys Admin
     
     A       new_config.json
     M       settings.conf
     ```
   - Example Output:
     ```
     TRANSACTION START abc1234
     SET_USER Sys Admin
     FILE_ADD new_config.json
     FILE_MOD settings.conf
     ```

2. **Legacy Interaction (`/home/user/send_to_legacy.exp`)**:
   Write an `expect` script that reads the translated commands from `stdin` (or a file passed as argument) and sends them to the legacy server.
   The legacy server will be accessible at `localhost:9999`.
   The expected interaction flow:
   - Server sends: `LOGIN:`
   - Script replies: `admin\n`
   - Server sends: `PASSWORD:`
   - Script replies: `legacy_secret\n`
   - Server sends: `READY>`
   - For each command line generated by the protocol converter, the script must send the command (plus `\n`) and wait for the server to reply `OK`.
   - After all commands are sent, the script must send `LOGOUT\n` and exit.
   - All console output of this `expect` session must be appended to `/home/user/logs/legacy_sync.log`.

3. **Log Rotation (`/home/user/logrotate.conf`)**:
   Create a logrotate configuration file for `/home/user/logs/legacy_sync.log`. It must:
   - Rotate the log daily.
   - Keep exactly 5 backups.
   - Compress old logs.
   - Ensure the file is not rotated if it is empty.

4. **Network Proxy (`socat`)**:
   The actual backend legacy server runs on `localhost:8888`. You must set up a TCP proxy using `socat` listening on `localhost:9999` that forwards connections to `localhost:8888`. Leave this `socat` process running in the background.

5. **Git Server and Hooks**:
   - Initialize a bare Git repository at `/home/user/central.git`.
   - Create a `post-receive` hook inside it.
   - For every push, the hook must extract the latest pushed commit using `git log -1 --name-status --pretty=format:"COMMIT:%H%nAUTHOR:%an"` and pass the output through your `/home/user/protocol_converter.py`, and then feed the resulting commands into your `/home/user/send_to_legacy.exp` script.
   - Ensure the hook is executable.

You do not have root access. Leave the proxy running and ensure all files are in their exact specified paths with appropriate permissions.
