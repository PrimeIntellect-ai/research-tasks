You are a platform engineer maintaining a CI/CD pipeline. We are introducing a new automated step to verify diagnostic audio artifacts generated by our UI testing suite. You need to build and deploy a local Python-based microservice that analyzes these audio files, adhering to strict performance and architectural constraints.

Your objective is to complete the following workflow:

1. **Compilation Step (Build Systems):**
   You have been provided with a raw C source file at `/app/extractor.c`. This tool extracts embedded steganographic transcripts and metadata from our test WAV files. 
   - Compile this C file into an executable located at `/app/bin/extractor`.
   - Ensure it is compiled with the `-O3` optimization flag.

2. **Configuration & Constraints (Constraint Satisfaction & Serialization):**
   - Create a configuration file at `/app/config/pipeline.json` with the following exact JSON structure:
     `{"max_duration_seconds": 300, "require_stego": true, "strict_mode": 1}`
   - Your Python service must deserialize this configuration on startup.

3. **API Service Implementation (Architecture & API Design):**
   - Write a Python API server (you may use `Flask`, `FastAPI`, or the standard `http.server`) that listens on `127.0.0.1:8080`.
   - Implement a `POST /analyze` endpoint.
   - When this endpoint receives a request, it must execute the compiled `/app/bin/extractor` binary against the target audio fixture located at `/app/ci_diagnostic.wav`.
   - The C binary outputs a custom key-value string format (e.g., `DURATION=45|TRANSCRIPT=hello world`). Your Python service must deserialize this string, validate that the duration does not exceed `max_duration_seconds` from your config, and then serialize the result into a JSON HTTP response.
   - The JSON response must look exactly like this:
     `{"status": "success", "duration": 45, "transcript": "hello world"}`
   - If the duration exceeds the constraint, return an HTTP 400 with `{"status": "error", "message": "Constraint violated"}`.

4. **Performance Benchmarking:**
   - Write a benchmarking script at `/app/benchmark.py` that sends 50 sequential POST requests to your `http://127.0.0.1:8080/analyze` endpoint.
   - Calculate the average latency in milliseconds.
   - Save the results to `/app/benchmark_results.json` in the format: `{"average_latency_ms": 12.5}`.

5. **Integration:**
   - Start your API server so it runs continuously in the background (e.g., using `nohup` or `&`).
   - Run your benchmark script to generate the results file.
   - Exit the terminal successfully. An automated verifier will then send real HTTP protocol requests to your running server to validate the architecture and logic.
