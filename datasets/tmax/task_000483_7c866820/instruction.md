You are an AI assistant helping a data scientist clean a messy time-series dataset generated by a flaky ETL job. The ETL job often retries on failure, resulting in duplicate records. It also occasionally mangles device names due to mixed character encodings and leaves gaps in the time-series.

Your task is to write a Python script `/home/user/pipeline.py` that processes `/home/user/dirty_telemetry.csv` and produces two output files: `/home/user/clean_telemetry.csv` and `/home/user/anomalies.csv`.

Here are the requirements for your data processing pipeline:

1. **Character Encoding Handling:**
   The `dirty_telemetry.csv` has three columns: `timestamp`, `device_hex`, and `value`. The `device_hex` column contains hex-encoded string bytes of the device name. Some were encoded using `utf-8` and some using `latin-1`. You must decode these hex strings to regular strings (try `utf-8` first, if it fails, fall back to `latin-1`). 

2. **Deduplication:**
   Due to ETL retries, there are exact duplicates for a given `timestamp` and decoded device name. Remove duplicates, keeping the first occurrence.

3. **Resampling and Gap-Filling:**
   For each unique device, set the `timestamp` as a DatetimeIndex. Resample the time-series to a 1-hour frequency (`1H`). Fill any gaps using forward-fill (`ffill`).

4. **Anomaly Detection:**
   After resampling, flag any rows where the `value` is greater than 100.0 as an anomaly.

5. **Outputs:**
   - Write the cleaned, resampled, and gap-filled data to `/home/user/clean_telemetry.csv`. It must have the columns: `timestamp`, `device`, `value`. (Sorted by `device`, then `timestamp`).
   - Write the anomalies to `/home/user/anomalies.csv`. It must have the same columns: `timestamp`, `device`, `value` and contain only the anomalous rows. (Sorted by `device`, then `timestamp`).

You may use `pandas` to accomplish this. Install it if necessary. Run your script to generate the final files.
