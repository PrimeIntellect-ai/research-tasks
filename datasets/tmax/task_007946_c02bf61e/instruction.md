You are a forensics analyst investigating a compromised Linux host. You have discovered a directory left behind by the attacker at `/home/user/forensics`. Inside this directory, there is an encrypted file named `exfiltrated.dat` that contains stolen data. 

Through earlier network traffic analysis, you know that the attacker's custom encryption tool uses a single-byte XOR cipher to encode the data before exfiltration. You also know that the plaintext log files generated by the attacker's system always begin with the exact string: `EVIDENCE_START`

Your objectives are:
1. **Reverse Engineer the Encryption:** Write a Python script to brute-force the 1-byte XOR key used to encrypt `/home/user/forensics/exfiltrated.dat` by looking for the known plaintext header.
2. **Decrypt the Payload:** Recover the full plaintext of the stolen data.
3. **Redact Sensitive Information:** The decrypted data contains stolen credit card numbers. You must locate all instances of credit card numbers formatted exactly as `XXXX-XXXX-XXXX-XXXX` (where X is a digit) and replace them with the string `[REDACTED_CC]`.
4. **Save the Evidence:** Output the decrypted and redacted text to a new file at `/home/user/forensics/clean_evidence.txt`.

Ensure your final file exactly preserves the original line breaks and text, modifying only the sensitive credit card numbers.
