You are a machine learning engineer preparing training data for a neural network surrogate model. The training data represents the total energy states of 50 different physical system configurations.

You have been given raw observational data generated by a parallel simulation. The data is located in `/home/user/raw_data/`. There are 50 text files named `sample_001.txt` through `sample_050.txt`. Each file contains 50,000 floating-point numbers (one per line) representing localized energy readings.

Because the legacy simulation accumulates these floating-point values in a non-deterministic parallel reduction order, naive summation produces slight precision errors that confuse the ML model. 

Your task is to write a script (in the language of your choice) to process these files and generate a clean, reproducible dataset at `/home/user/ml_dataset.csv`.

Requirements for your script:
1. **Parallelism:** You must process the files in parallel (using multiple threads or processes) to speed up data preparation.
2. **Reproducible Reduction:** To eliminate non-reproducible floating-point reduction errors, you must use an exact mathematical summation algorithm (such as Kahan summation or an equivalent high-precision built-in library function) to sum the 50,000 values in each file.
3. **Analytical Validation:** The physical system has a known analytical limit. The theoretical exact total energy for a given `sample_X.txt` is analytically defined as exactly `X * 1000.5` (where `X` is the integer sample ID). You must validate your computed sum against this analytical solution. If the absolute difference between your exact computed sum and the analytical solution is greater than `1e-7`, you must assume the simulation step was corrupted and completely exclude that sample from your final dataset.
4. **Data Reshaping:** Output a final CSV file at `/home/user/ml_dataset.csv` with a header row `sample_id,total_energy`. 
   - `sample_id` should be the integer ID (e.g., 1 for `sample_001.txt`).
   - `total_energy` must be the computed exact sum formatted to exactly 4 decimal places.
   - The rows must be sorted by `sample_id` in ascending order.
