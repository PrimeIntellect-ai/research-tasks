You are tasked with organizing project files by building a robust log rotation and archiving system. 

Currently, a chaotic background process is actively writing multi-line log records (like Java stack traces) to a single log file at `/home/user/active_logs/app.log`. The process writes very fast, and you must rotate this log file into an archive directory (`/home/user/archive/`) without losing any records or corrupting active writes.

Your requirements:
1. **Understand the Target Format**: Inspect the image located at `/app/spec.png`. It contains a short text snippet specifying a "Magic Header" (8 characters long). You must use OCR (e.g., `tesseract`) to read it.
2. **Multi-line Record Parsing**: The logs in `app.log` start with `[YYYY-MM-DD HH:MM:SS] <LEVEL>`. Many records (especially ERRORs) span multiple lines. A record only ends when the next `[` timestamp begins, or at the end of the file.
3. **Atomic Rotation & Locking**: Write a Python script `/home/user/rotator.py` that safely extracts complete records from `app.log`, clears the extracted records from the active file without losing new appends (using `fcntl` file locking or atomic temp-file replacement), and organizes them.
4. **Custom Compression**: Compress the extracted multi-line records. The final archived files must be placed in `/home/user/archive/` (e.g., `app_rotated_1.arc`). Each `.arc` file MUST be a custom compressed binary file:
   - The first 8 bytes must be exactly the Magic Header you extracted from the image.
   - The remainder of the file must be the raw standard `gzip` compressed bytes of the extracted log text.
5. **Execution**: Run your rotator script. You can run it multiple times or in a loop while the writer is active.

Your success will be measured by a quantitative metric: the ratio of valid, uncorrupted multi-line records successfully archived divided by the total number of records generated by the background process. You must achieve a recovery ratio of >= 0.95.

Create your script, run it to archive the currently accumulated and streaming logs, and ensure the `/home/user/archive/` directory is populated with your `.arc` files.
