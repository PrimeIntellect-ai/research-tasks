You are a data analyst working with a dump of international user reviews. 

The directory `/home/user/reviews/` contains multiple CSV files (`data_1.csv` through `data_5.csv`). These files were generated by different regional servers and are saved in different, unknown character encodings (e.g., UTF-8, ISO-8859-1, Shift_JIS, UTF-16, CP1251). 

The CSV files have the following columns:
`Date,Username,Review_Text`

Your task is to write a Python script that performs the following pipeline:
1. **Parallel Processing**: Read and process the CSV files in parallel using Python's `concurrent.futures` or `multiprocessing` modules.
2. **Encoding Handling**: Programmatically detect or handle the different character encodings of the files so they can be read correctly without errors. You may install and use libraries like `chardet` if needed.
3. **Unicode Normalization**: Extract the `Review_Text` column and normalize all text using Unicode **NFKC** normalization.
4. **Aggregation**: Calculate the length (number of characters) of each normalized review. Calculate the average normalized review length for each `Date`.
5. **Changepoint Detection**: Sort the aggregated data chronologically by Date. A "changepoint" is defined as any Date where the average normalized review length is **strictly greater than 2.0 times** the average normalized review length of the *immediately preceding date* in the sorted chronological sequence.
6. **Output**: Save the dates of these changepoints as a JSON array of strings (format: `["YYYY-MM-DD", ...]`) into the file `/home/user/changepoints.json`.

Make sure your script correctly normalizes the text before taking the length, as this will affect the character count for certain multi-language characters. Ensure you handle the file encodings gracefully. 

When you have created `/home/user/changepoints.json`, your task is complete.
