You are acting as a security researcher investigating a suspicious, proprietary logging infrastructure found on a compromised system. 

The compromised system runs a multi-service architecture located in `/app/services/`:
1. `log_receiver`: A binary listening on `127.0.0.1:9001` that ingests raw log payloads.
2. `db_writer`: A backend service on `127.0.0.1:9002` that writes parsed logs.
3. `health_monitor`: A service on `127.0.0.1:9003` tracking the uptime of the other two.

Your goal is to build a robust Bash-based sanitization filter to block malicious payloads that exploit a zero-day in `log_receiver`, while allowing benign payloads to pass through. 

To achieve this, you must complete the following stages:

**Stage 1: Forensic Recovery & Timeline Analysis**
We have captured a loopback filesystem image of the attacker's scratch directory: `/app/evidence.img`. During the breach, the attacker dropped a temporary stage-2 payload and subsequently deleted it. 
- Extract the deleted file from `/app/evidence.img`. It contains the base hexadecimal signature of the exploit.
- Inspect the log files generated by the three services located in `/var/log/app_services/`. Reconstruct the exact timeline of the attack to understand how the payloads flow from `log_receiver` to `db_writer`. 

**Stage 2: Fuzzing the Suspicious Binary**
We know `log_receiver` crashes when specific bytes wrap the recovered signature. 
- Write a simple Bash fuzzer to send variations of the recovered signature to `127.0.0.1:9001`.
- Identify the exact byte-padding and prefix requirements that trigger the crash (monitor the `health_monitor` logs to see when `log_receiver` restarts).

**Stage 3: Developing the Sanitizer (Adversarial Corpus Verification)**
Under `/app/corpus/`, there are two directories:
- `/app/corpus/clean/`: Contains 500 benign log payloads.
- `/app/corpus/evil/`: Contains 500 malicious log payloads utilizing variations of the exploit.

Write a Bash script at `/home/user/filter.sh` that takes a single file path as its first argument (`$1`). 
- If the file contains a benign payload, your script must exit with status code `0`.
- If the file contains a malicious payload, your script must exit with status code `1`.

Your script will be tested against both corpora. You must accurately classify 100% of both the clean and evil corpora.

Constraints:
- Your filter must be written in Bash (`#!/bin/bash`).
- You may use standard Unix utilities (`grep`, `awk`, `sed`, `hexdump`, etc.) inside your script.
