You are acting as a DevSecOps engineer enforcing security policies on a legacy authentication microservice. You have been provided with the source code of the service and some application logs. 

Your objective is to audit the code, demonstrate its vulnerabilities, and implement a data redaction policy using bash.

You have the following files:
1. `/home/user/server.py`: A Python web server implementing a basic authentication API.
2. `/home/user/app.log`: A log file generated by this server in production, which currently leaks sensitive PII.

Perform the following tasks:

1. **CWE Identification**:
Audit `/home/user/server.py` and identify three primary vulnerabilities. Write the corresponding CWE IDs (e.g., `CWE-123`) to `/home/user/cwes.txt`, one per line. Specifically look for:
- An authentication bypass mechanism using a custom header.
- The presence of a hardcoded secret token.
- The logging of sensitive PII to standard output/logs.

2. **Exploit Crafting**:
Write a bash script at `/home/user/exploit.sh` that successfully exploits the authentication bypass vulnerability in the local service. 
- You may run `python3 /home/user/server.py &` to start the server on port 8080 for your testing.
- Your script must use `curl` to send a request to `http://127.0.0.1:8080/login` bypassing the normal credentials by utilizing the flaw found in the code.
- The server will return a JSON response containing a `secret_token` if successful. Your `exploit.sh` script must print ONLY the value of this secret token to standard output.

3. **Sensitive Data Redaction**:
Create a bash script at `/home/user/redact.sh` that reads a file provided as its first argument and redacts sensitive data using `sed`, `awk`, or standard bash utilities.
- Replace all Social Security Numbers (format: `XXX-XX-XXXX`) with the exact string `[REDACTED_SSN]`.
- Replace all email addresses with the exact string `[REDACTED_EMAIL]`.
- Apply your script to the provided log file by running: `/home/user/redact.sh /home/user/app.log > /home/user/clean.log`.

Ensure all scripts (`exploit.sh` and `redact.sh`) are executable (`chmod +x`).
