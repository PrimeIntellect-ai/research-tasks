You are tasked with handling the credential rotation migration for a legacy internal service. The legacy authentication system uses an undocumented, stripped binary to generate authentication tokens, and we need to replace it with a modern HTTP service. 

Follow these requirements precisely:

1. **Log Analysis and Credential Recovery**
   - You have access to `/var/log/auth_sys.log`, which contains recent login attempts. 
   - Parse the log to identify the users who have *successfully* authenticated. The log entries contain MD5 hashes of their passwords.
   - Using the dictionary provided at `/opt/wordlist.txt`, crack the MD5 hashes to recover the plaintext passwords for these active users.
   - Save the recovered credentials as a JSON file at `/app/recovered_creds.json` in the format: `{"username1": "password123", "username2": "..."}`.
   - For security reasons, ensure `/app/recovered_creds.json` has exactly `0600` permissions.

2. **Legacy Token Analysis**
   - The old system uses a stripped binary located at `/app/legacy_token_gen`. 
   - When run via `./legacy_token_gen <username> <password>`, it outputs a hexadecimal legacy token.

3. **New Auth & Rotation Proxy**
   - Write and start an HTTP service (in any language of your choice) listening on `127.0.0.1:8000`.
   - The service must read `/app/recovered_creds.json` on startup to know the current valid credentials.
   - Implement the following two endpoints:
     
     **A. `POST /rotate`**
     - Accepts a JSON body: `{"username": "<username>", "new_password": "<new_pass>"}`
     - Requires an HTTP header: `X-Legacy-Token: <token>`
     - **Validation:** The service must verify that the `X-Legacy-Token` matches the token generated by `/app/legacy_token_gen` for that user's *current* (old) password. 
     - If the token is valid, update the user's password to `new_password` in memory and return HTTP status `200 OK`.
     - If the token is invalid or missing, return HTTP status `401 Unauthorized`.

     **B. `GET /verify`**
     - Requires standard HTTP Basic Authentication.
     - If the provided username and password match the user's *current* password (the recovered one, OR the newly rotated one if they have used the `/rotate` endpoint), return HTTP status `200 OK` with a JSON body `{"status": "ok"}`.
     - If the credentials do not match, return HTTP status `401 Unauthorized`.

Keep the service running in the foreground or background so it can be tested. You may install any necessary packages or language runtimes.
