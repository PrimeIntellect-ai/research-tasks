You are an MLOps engineer tasked with building a resilient ETL pipeline to process and track artifact metrics from simulated machine learning experiments. 

We have raw experiment logs generated by training runs. These logs are stored as JSON Lines (.jsonl) files in `/home/user/raw_logs/`. Some log entries are corrupted, contain invalid types, or have physically impossible values (like negative loss).

Your task is to build a bash-orchestrated pipeline and a Python ETL script that:
1. Enforces a strict data schema on the logs.
2. Cleans the data by dropping invalid rows.
3. Calculates the Exponential Moving Average (EMA) of the validation loss for each experiment to test numerical accuracy.
4. Outputs a consolidated summary artifact.

### Requirements:

**1. Data Schema Enforcement**
Valid records must meet ALL the following criteria. Any row failing these should be completely ignored (dropped):
- `experiment_id`: Must be a string.
- `epoch`: Must be an integer >= 0.
- `val_loss`: Must be a float (or convertible to float) and strictly > 0.

**2. Data Processing & Numerical Accuracy**
For each valid experiment, calculate the EMA of `val_loss` across epochs. 
- You MUST sort the valid records for each experiment strictly by `epoch` in ascending order before calculating the EMA.
- The EMA formula to use is: 
  - $EMA_0 = val\_loss_0$ (for the first valid epoch chronologically)
  - $EMA_t = \alpha \cdot val\_loss_t + (1 - \alpha) \cdot EMA_{t-1}$
- Use $\alpha = 0.2$.

**3. Output Format**
Create a Python script `/home/user/process_metrics.py` that reads all `.jsonl` files in `/home/user/raw_logs/`, performs the above logic, and writes the final EMA of the validation loss for each experiment to `/home/user/artifacts/summary.json`.
The output JSON must be a simple dictionary mapping the `experiment_id` to its final EMA, rounded to exactly 4 decimal places.
Example:
```json
{
  "EXP_A": 0.8520,
  "EXP_B": 1.1234
}
```

**4. Pipeline Orchestration**
Write a bash script `/home/user/run_pipeline.sh` that:
- Creates the `/home/user/artifacts/` directory if it doesn't exist.
- Sets up a Python virtual environment at `/home/user/venv`.
- Installs any necessary dependencies (e.g., `pydantic`, though standard library is also fine if you prefer).
- Runs the `process_metrics.py` script.

Make sure your shell script is executable. You should execute `./run_pipeline.sh` to produce the final `summary.json` before completing the task.
