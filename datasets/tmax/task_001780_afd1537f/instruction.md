You are assisting a data researcher who is organizing a highly unconventional dataset. The researcher has received a video file containing a visual log of network topology data. 

Your task is to build a data extraction pipeline and a queryable Rust graph database service.

**Step 1: Data Extraction**
A video file is located at `/app/dataset_log.mp4`. It contains a sequence of frames, some of which feature QR codes. Each QR code encodes a directed edge in a network graph in the following format:
`GRAPH_EDGE;<source_node>;<target_node>;<weight>`
(e.g., `GRAPH_EDGE;Alpha;Beta;15`)

You must:
1. Extract the frames from the video using `ffmpeg`.
2. Extract the text from the QR codes using `zbarimg`.
3. Process the extracted text into a clean CSV dataset of edges.

**Step 2: Rust Graph Query Service**
Write and run a Rust HTTP service (in `/home/user/graph_api`) that acts as a query engine for this dataset.
1. The service must parse the extracted CSV and build an efficient in-memory index for graph traversal (e.g., an adjacency list).
2. It must listen continuously on `127.0.0.1:8080`.
3. It must expose a `POST /shortest_path` endpoint.
4. The endpoint must require an `Authorization` header with the exact value: `Token research-2024`. Requests without this or with an incorrect token must return a `401 Unauthorized`.
5. The request payload will be JSON matching this schema:
   `{"start_node": "string", "end_node": "string"}`
6. The service must compute the shortest path (lowest total weight) between the two nodes using Dijkstra's algorithm or similar.
7. The response must be JSON matching this schema exactly:
   `{"path": ["start_node", "intermediate_node", ..., "end_node"], "total_weight": integer}`
8. If no path exists, return a `404 Not Found` with `{"error": "No path exists"}`.

**Constraints & Guidelines:**
- Use standard bash tools (`ffmpeg`, `zbarimg`, `awk`, etc.) for the extraction pipeline.
- Write the service in Rust. You may use any standard crates (e.g., `axum`, `tokio`, `serde`, `serde_json`, `petgraph`).
- Do not hardcode the graph data into your Rust code; it must read the file generated by your extraction pipeline.
- Once the service is running, leave it running in the background so the researcher's automated test suite can query it. Create a log file at `/home/user/service_ready.log` containing the text "SERVICE_UP" when your server is bound to the port and ready to receive requests.
