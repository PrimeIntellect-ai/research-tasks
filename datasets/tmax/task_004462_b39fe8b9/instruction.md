You are tasked with completing and configuring a local artifact management system that curates uploaded ELF binaries. The system consists of multiple services that need to be connected, and a core parsing script you must write from scratch.

The setup is located in `/home/user/artifact_manager`.

### 1. Service Configuration
The system uses three components:
- A Flask Web Application (`/home/user/artifact_manager/app.py`) listening on port 5000 for file uploads.
- A Redis instance acting as a metadata queue (running on `127.0.0.1:6379`).
- A Background Worker (`/home/user/artifact_manager/worker.py`).

Your task:
- The Flask app currently uses a mock in-memory queue. Modify its configuration file (`/home/user/artifact_manager/config.ini`) to use the Redis instance instead. Set the `queue_type` to `redis` and ensure the connection string points to `redis://127.0.0.1:6379/0`.
- Start the services using the provided `/home/user/artifact_manager/start_services.sh` script. Ensure they remain running.

### 2. ELF Parser Implementation
You must implement the core ELF parsing logic in Python. Create a script at `/home/user/artifact_manager/elf_parser.py`.
- The script must take exactly one command-line argument: the absolute path to an ELF binary.
- It must parse the ELF file and print exactly one line to standard output in the following format:
  `<Machine Architecture>|<Entry Point in Hex>|<Size of .text section in bytes>`
- Example output: `EM_X86_64|0x401050|1420`
- If the `.text` section is missing, print `0` for its size.
- You may use the `pyelftools` library, which is already installed.
- Ensure the script is executable (`chmod +x`).

### 3. Workflow Integration
The background worker tails a multi-line log file (`/home/user/artifact_manager/logs/upload.log`) generated by the Flask app. When a new ELF file is uploaded to `/home/user/artifact_manager/uploads`, the worker runs your `elf_parser.py`.
Modify the worker (`worker.py`) so that after it receives the parsed string, it creates a symbolic link to the uploaded binary in `/home/user/artifact_manager/curated/<Architecture>/<Original Filename>`. You will need to extract the `<Architecture>` from your parser's output and create the directory if it doesn't exist.

Your final system must successfully process end-to-end uploads: receiving an ELF file via the Flask app, logging it, popping the event from Redis, parsing it with your script, and creating the correct symlink under the `curated/` directory.
