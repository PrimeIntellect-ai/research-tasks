You are acting as a Level 3 Support Engineer. A customer has escalated an issue with their numerical simulation pipeline, which runs inside a Linux environment. The pipeline consists of a Python script that occasionally crashes with stack traces, fails to converge to an equilibrium state, and occasionally produces statistical anomalies in the generated logs.

Your task is to diagnose and fix the issues in the pipeline.

Phase 1: Statistical Anomaly Investigation
The customer has provided a historical log file located at `/home/user/sim_runs.log`. This file contains the results of 1000 simulation runs.
One of the runs experienced a statistical anomaly where the `MeanEnergy` was wildly out of distribution (greater than 1000.0), whereas normal runs hover between 10.0 and 20.0.
1. Inspect `/home/user/sim_runs.log`.
2. Find the `RunID` associated with this anomalous `MeanEnergy`.
3. Write ONLY the exact `RunID` string (e.g., `RUN_123`) to a new file at `/home/user/anomaly.txt`.

Phase 2: Core Logic Debugging (Boundary Condition & Convergence)
The customer's main simulation script is located at `/home/user/simulation.py`. Currently, if you try to run it via `python /home/user/simulation.py --run-all`, it fails. 

There are two primary bugs in the script:
1. **Boundary Condition / Off-By-One Error:** The `run_diffusion(grid_size)` function crashes with an `IndexError` due to an incorrect loop boundary when computing the 1D spatial derivative.
2. **Convergence Failure:** The `find_equilibrium(init_val)` function is supposed to use a simple gradient descent step to find the stable state (which is mathematically known to be 50.0 for this system). However, it diverges to infinity because of a sign error in the update rule.

Your job:
1. Edit `/home/user/simulation.py` to fix the off-by-one boundary condition in `run_diffusion`.
2. Edit `/home/user/simulation.py` to fix the convergence failure in `find_equilibrium` so it successfully converges to `50.0`.
3. Run the repaired script using `python /home/user/simulation.py --run-all`. 

If your fixes are correct, the script will execute successfully without exceptions and automatically write a JSON file to `/home/user/success.json` containing the finalized computed states.

Success is defined by:
1. `/home/user/anomaly.txt` containing the correct anomalous RunID.
2. `/home/user/success.json` existing and containing the mathematically correct values generated by the fixed script.
