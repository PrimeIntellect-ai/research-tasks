You are a storage administrator tasked with managing disk space on a logging server. You have received a compressed archive of mixed log files from various applications, and you need to safely process them to optimize storage.

Your goal is to write and execute a Bash script at `/home/user/process_logs.sh` that performs the following steps:

1. **Extraction**: Extract the contents of the archive `/home/user/incoming_logs.tar` into a new directory `/home/user/processing`. The archive contains a nested directory structure with various files.
2. **Path Manipulation and Filtering**: Find all files within `/home/user/processing` (including all subdirectories) that have the `.log` extension.
3. **Safe Splitting**: For each `.log` file found:
   - You must ensure no other process is writing to the file while you read it. Obtain a shared file lock using `flock` on the file descriptor before reading/splitting it.
   - Split the file into smaller chunks of exactly 50 lines each.
4. **Bulk Renaming**: The chunks generated by the split command should be saved in the same directory as the original file. They must be renamed to match the format: `<original_filename_without_extension>_chunk_<suffix>.txt`. For example, if the original file is `app1.log` and split produces `xaa` and `xab`, the resulting files should be named `app1_chunk_aa.txt` and `app1_chunk_ab.txt`.
5. **Archiving**: Once all `.log` files have been safely locked, split, and the chunks renamed, find all the newly created `*chunk*.txt` files and package them into a single, flat compressed archive at `/home/user/final_chunks.tar.gz`. "Flat" means the archive should not contain any directory structure; all chunk files should be at the root of the archive.

**Verification Requirements**:
