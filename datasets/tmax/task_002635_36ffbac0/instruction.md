You are an incident responder and network engineer investigating a compromised SSH infrastructure. 

We have isolated the affected environment in the `/app` directory. The attackers replaced our SSH authentication command with a malicious compiled binary, and we are currently capturing active attack traffic against it.

Your objective is to re-establish the environment, reverse engineer the malicious binary to understand the backdoor, and build a detection mechanism.

**Environment Details:**
Inside `/app`, you will find:
1. `ssh/`: Contains an `sshd_config` file and host keys.
2. `bin/auth_hook`: A compiled, stripped Linux ELF binary used by the attackers for SSH authentication.
3. `traffic_generator.py`: A local service that simulates the captured network traffic by making continuous SSH connection attempts to port 2222.
4. `start_services.sh`: A script that starts the custom `sshd` instance (on port 2222) and the `traffic_generator.py`.

**Step 1: SSH Service Hardening & Setup**
The `sshd_config` in `/app/ssh/` is currently broken and refuses to start. 
- Fix the configuration so that `sshd` runs on port 2222 as the current user.
- Configure it to use `/app/bin/auth_hook` as the `AuthorizedKeysCommand` (with `AuthorizedKeysCommandUser` set to the current user).
- Ensure SSH logs are directed to `/home/user/ssh_logs/auth.log` (you may need to configure SyslogFacility or hook a local syslog daemon listening in user-space, e.g., using `rsyslog` local config or a python mock syslog server).
- Run `/app/start_services.sh` to begin the traffic simulation.

**Step 2: Reverse Engineering**
Analyze `/app/bin/auth_hook`. This binary receives the SSH username as an argument. The attackers implemented a backdoor: if a specific username is provided, it bypasses normal checks and grants access. Reverse engineer the binary to discover this exact backdoor username string.

**Step 3: Log Parsing and Detection Script**
Write a script at `/home/user/detector.py` that takes a single command-line argument (the path to an SSH log file) and analyzes the logs to find IP addresses that successfully authenticated using the backdoored username.
- Your script must print the malicious IP addresses to standard output, exactly one IP per line, with no other text.
- The script should be able to handle standard Linux `auth.log` formats generated by `sshd`.

**Evaluation:**
Your `detector.py` script will be evaluated against a massive, held-out production log file containing hundreds of thousands of events with varying levels of obfuscation. Your script's output will be compared against the ground truth of known malicious IPs.
To pass this task, your detection script must achieve an F1-score of **>= 0.95** on the held-out test set.
