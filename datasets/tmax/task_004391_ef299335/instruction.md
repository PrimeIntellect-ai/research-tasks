I am a researcher organizing a massive video dataset, and I need a robust tool to process configuration files that define frame extraction jobs. Because these configuration files are often generated by external, untrusted sources, I need a C program that rigidly validates them before executing any commands.

Write a C program at `/home/user/dataset_filter.c` and compile it to `/home/user/dataset_filter`.

Your program must take exactly one command-line argument: the path to a configuration file.

The configuration file is a plain text file containing exactly four key-value pairs (one per line, separated by a colon and a single space):
`dataset_name: <string>`
`output_archive: <filename>`
`start_frame: <int>`
`end_frame: <int>`

**Validation Rules (Reject with exit code 1 if any are violated):**
1. The file must contain all four keys exactly as named above.
2. `output_archive` must be a valid filename containing ONLY alphanumeric characters, underscores, and ending exactly with `.tar`. It must not contain `/`, `\`, or `..`.
3. `start_frame` and `end_frame` must be integers.
4. `start_frame` must be >= 0.
5. `end_frame` must be strictly greater than `start_frame`.
6. `end_frame` must not exceed the total number of frames in the provided video dataset.

**Processing (If valid):**
1. Read the configuration file using `mmap` (memory-mapped I/O). Do not use standard `fopen`/`fread` or `ifstream` for reading the configuration file contents.
2. The source video is located at `/app/dataset_video.mp4`. You must extract the frames from `start_frame` to `end_frame` (inclusive of `start_frame`, exclusive of `end_frame`) as JPEG images. You may use `ffmpeg` via `system()` or `popen()`. Assume the video runs at exactly 25 FPS and has no variable frame rate issues.
3. Extracted frames must be temporarily stored in a securely created temporary directory.
4. Create an uncompressed tar archive containing these frames. The frames inside the archive should be named `frame_0000.jpg`, `frame_0001.jpg`, etc., relative to the start of the extraction (i.e., the first extracted frame is `frame_0000.jpg`).
5. Write the final tar archive to the current working directory using the name specified in `output_archive`. To prevent partial writes from crashing the pipeline, you must write the tar file to a temporary file first, and then atomically rename it to the final `output_archive` name.
6. Clean up the temporary directory.
7. Exit with code 0.

Ensure your C program is compiled and ready to be tested against a large corpus of both valid and malicious configuration files.
