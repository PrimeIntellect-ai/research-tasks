You are a performance engineer tasked with optimizing and replacing a legacy Monte Carlo simulation tool used in our computational geometry pipeline.

We have a stripped legacy binary located at `/app/vol_estimator`. This tool estimates the volume of a Steinmetz solid (the intersection of two cylinders of radius $R$ intersecting at right angles, specifically $x^2 + y^2 \le R^2$ and $x^2 + z^2 \le R^2$).
The tool takes three arguments: `./vol_estimator <seed> <N> <R>`
- `seed`: Integer seed for the PRNG.
- `N`: Number of Monte Carlo samples.
- `R`: Radius of the cylinders (floating point).

The binary outputs a single integer: the number of generated points that fall strictly inside or on the boundary of the intersection.

We need to replace this black-box binary with a high-performance C program that produces **bit-exact identical results**. From my preliminary reverse-engineering of the legacy binary, I found it uses a specific custom deterministic sequence instead of the standard libc `rand()`:
1. It uses an LCG for its random number generator: $X_{n+1} = (1103515245 \times X_n + 12345) \pmod{2^{31}}$.
2. The state $X_0$ is initialized to `seed`.
3. For each of the `N` iterations, it generates $x, y, z$ coordinates in that exact order.
4. A coordinate is generated by drawing the next LCG integer and mapping it to the range $[-R, R]$ using double precision arithmetic: `coord = -R + (2.0 * R * (double)lcg_val) / 2147483648.0;`.

**Your tasks:**

1. **Analytical Validation & Visualization:** Write a Python script `/home/user/validate.py` that calculates the analytical volume of the Steinmetz solid ($V = \frac{16}{3} R^3$). The script should run the legacy binary `/app/vol_estimator` for $R=5.0$, with a fixed seed of `42`, and vary $N$ from $1000$ to $10000$ in steps of $1000$. For each $N$, calculate the estimated volume ($V_{est} = \frac{\text{count}}{N} \times (2R)^3$) and plot the absolute error between $V_{est}$ and the analytical volume. Save the plot to `/home/user/convergence.png`.

2. **C Implementation:** Write a C program `/home/user/fast_vol.c` that exactly replicates the logic of the legacy binary.
   - It must take the same command-line arguments: `<seed> <N> <R>`.
   - It must print only the integer count of points inside the intersection, followed by a newline.
   - Compile it to an executable at `/home/user/fast_vol` (e.g., using `gcc -O3 /home/user/fast_vol.c -o /home/user/fast_vol`).

Your compiled C executable must perfectly match the output of the legacy binary for any valid combination of inputs.
