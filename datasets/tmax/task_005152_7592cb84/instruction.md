You are a data engineer tasked with building an ETL pipeline script that normalizes and reshapes high-frequency IoT sensor logs.

Your objective is to create a Python script at `/home/user/etl_processor.py` that reads wide-format CSV data from standard input (`stdin`) and outputs processed long-format JSON lines to standard output (`stdout`).

**Data Processing Requirements:**
1. **Input Format:** A CSV stream (via stdin) with headers. The first column is `raw_time`. Subsequent columns represent individual sensors (e.g., `sensor_alpha`, `sensor_beta`). 
   Example row: `2023-08-15T14:35:49Z,"[VAL: 42.1] [STATUS: OK]","[VAL: -3.2] [STATUS: ERR]"`
2. **Timestamp Alignment:** Parse the `raw_time` column and floor/truncate it to the nearest minute (e.g., `2023-08-15T14:35:49Z` becomes `2023-08-15T14:35:00Z`). All timestamps are UTC.
3. **Wide-to-Long Reshaping:** Unpivot the sensor columns so each row represents a single sensor reading. Process sensors in alphabetical order of their column names.
4. **Tokenization & Normalization:** For each sensor reading, extract the floating-point value and the status code. The format is always `[VAL: <float>] [STATUS: <string>]`.
5. **Constraint-based Validation:** Drop the reading entirely if:
   - The status is NOT exactly `OK`.
   - The parsed float value is strictly less than `0.0`.
6. **Output Format:** For each valid reading, emit a single JSON object on a new line (JSONL). The object must have strictly these keys: `{"ts": "<aligned_timestamp_string>", "sensor": "<sensor_name>", "val": <float_value>}`.
