You are tasked with building a high-performance C++ data pipeline to join, validate, and clean user datasets and model predictions. 

As a data scientist, you frequently need to ensure that the data fed into downstream analytics is strictly formatted and logically valid. 

You have two input files located in your home directory:
1. `/home/user/users.csv` - Contains user demographic data.
   Columns: `user_id` (integer), `age` (integer), `region` (string)
2. `/home/user/predictions.csv` - Contains a machine learning model's output for these users.
   Columns: `user_id` (integer), `score` (float), `class_label` (integer)

Write a C++ program at `/home/user/clean_data.cpp`, compile it, and run it to perform the following tasks:

1. **Multi-source Join:** Perform an inner join on the two datasets using `user_id`. Records that do not exist in both datasets should be silently ignored and dropped.
2. **Schema & Model Validation:** For the joined records, enforce the following data schema and validation rules:
   - `age` must be between 18 and 120 (inclusive).
   - `score` must be between 0.0 and 1.0 (inclusive).
   - `class_label` must be exactly `0` or `1`.
3. **Valid Output:** Write all records that pass **ALL** validation checks to `/home/user/cleaned_output.csv`.
   - The output must include a header: `user_id,age,region,score,class_label`
   - The rows must be sorted by `user_id` in ascending order.
4. **Error Logging:** For joined records that fail **ANY** validation check, write their `user_id` to `/home/user/validation_errors.txt`.
   - Write exactly one `user_id` per line.
   - The IDs must be sorted in ascending order.
   - Do not include header lines in this file.

Both output files must be generated by your compiled C++ program.
