You are an MLOps engineer tasked with verifying the reproducibility of dimensionality reduction artifacts generated by various experiment runs. 

The principal data scientist left a voice memo detailing the exact pipeline parameters used for the canonical baseline. The memo is located at `/app/audio/memo.wav`. 
You need to transcribe or listen to this audio file to extract the pipeline specifications (specifically: the scaling method, the dimensionality reduction algorithm, the number of components, and the random seed).

You are provided with the raw input dataset at `/app/data/raw_features.csv`.

Your task is to write a Python CLI script at `/home/user/filter_artifacts.py` that validates experiment artifacts.
An artifact is a CSV file containing an embedding matrix (rows represent samples, columns represent components). 
An artifact is considered **valid** if and only if its embeddings match the output of the canonical reproducible pipeline described in the audio memo (using `scikit-learn` defaults for any unspecified parameters, applied to the entirety of `/app/data/raw_features.csv`), with a numerical tolerance of `1e-5` for floating-point differences.

Your script must conform to the following interface:
1. Accept a single argument `--input` specifying the absolute path to an artifact CSV file.
2. Load the specified artifact and compare it against the canonical reference embeddings.
3. Print exactly `ACCEPT` to standard output (and exit with code 0) if the artifact is valid.
4. Print exactly `REJECT` to standard output (and exit with code 1) if the artifact is invalid (e.g., wrong dimensions, incorrect values, corrupted data).

Example invocation:
`python /home/user/filter_artifacts.py --input /some/path/artifact_1.csv`

Requirements:
- You must create the Python script `/home/user/filter_artifacts.py`.
- You will need to install any necessary tools to transcribe the audio file and libraries to implement the pipeline (e.g., `scikit-learn`, `pandas`, `numpy`, `openai-whisper` or similar). 
- Ensure your pipeline strictly adheres to the parameters mentioned in the audio.
