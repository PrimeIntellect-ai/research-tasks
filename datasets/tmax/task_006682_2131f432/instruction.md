You are a monitoring specialist tasked with setting up a custom alerting agent for a staged deployment. As part of our deployment pipeline, we need a lightweight C++ monitor that checks if the new stage's service port is active, and if not, triggers an email alert to the mailing list via a local SMTP relay.

Perform the following steps:

1. **Environment Setup:**
   Create a shell profile script at `/home/user/monitor_env.sh` that exports the following environment variables:
   - `MONITOR_PORT=9090` (The port of the staged deployment service)
   - `SMTP_PORT=2525` (The port of our local mock SMTP server)
   - `ALERT_EMAIL=oncall@staged-deploy.local` (The mailing list address)

2. **Mock SMTP Server:**
   A mock SMTP server script is located at `/home/user/mock_smtp.py`. Start it in the background using `python3 /home/user/mock_smtp.py &`. It will listen on the port specified by `SMTP_PORT` and append all received email data to `/home/user/email_alerts.log`.

3. **C++ Monitoring Agent:**
   Write a C++ program at `/home/user/monitor.cpp` and compile it to `/home/user/monitor`.
   The program must:
   - Read the `MONITOR_PORT`, `SMTP_PORT`, and `ALERT_EMAIL` environment variables.
   - Attempt to establish a TCP connection to `127.0.0.1` on `MONITOR_PORT`.
   - Because port 9090 is inactive, the connection will fail. Catch this failure.
   - Upon failure, establish a TCP connection to `127.0.0.1` on `SMTP_PORT`.
   - Send the raw SMTP commands to deliver an email. The commands must follow this exact sequence (separated by `\r\n`):
     ```
     HELO localhost
     MAIL FROM:<monitor@localhost>
     RCPT TO:<[value of ALERT_EMAIL]>
     DATA
     Subject: Alert: Port Down
     
     Port [value of MONITOR_PORT] is unresponsive.
     .
     QUIT
     ```
   *(Note: ensure you read the SMTP server's responses or simply introduce a small delay between sending commands if necessary, though the mock server accepts pipelined commands).*

4. **Execution:**
   Source your environment script and execute `/home/user/monitor`. 

To succeed, the final `/home/user/email_alerts.log` must contain the exact subject and body generated by your C++ program.
