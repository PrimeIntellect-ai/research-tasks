I am a researcher organizing a growing dataset of environmental readings. I have a multi-line text log file generated by my instruments, a configuration file detailing what data is currently relevant to my research, and an existing backup file containing previously extracted records. 

I need you to write and execute a Python script that parses the multi-line log, filters it based on the configuration, identifies only the *new* records that haven't been backed up yet (acting as an incremental backup), and appends them to my backup file in a specific format.

Here are the details:

1. **Log File (`/home/user/dataset/instrument.log`)**:
   Contains multi-line records separated by a line containing exactly `---`. Each record has key-value pairs separated by `: `.
   Example:
   ```
   ---
   RecordID: 001
   Date: 2023-10-01
   SensorType: Temperature
   Value: 24.5
   ---
   RecordID: 002
   ...
   ```

2. **Configuration File (`/home/user/dataset/criteria.ini`)**:
   An INI file that specifies which records I care about.
   ```ini
   [Filters]
   valid_sensors = Temperature, Humidity
   min_value = 0.0
   ```

3. **Existing Backup (`/home/user/backup/archive.jsonl`)**:
   A JSON Lines (JSONL) file where each line is a valid JSON object representing a previously saved record.
   
**Your Task:**
Write a Python script (save it anywhere, e.g., `/home/user/process_data.py`) and run it to do the following:
1. Parse `criteria.ini` to get the `valid_sensors` (comma-separated list) and `min_value` (float).
2. Parse `instrument.log` into individual records.
3. Filter the records: Keep a record ONLY if its `SensorType` is in `valid_sensors` AND its `Value` (as a float) is strictly greater than `min_value`.
4. Implement incremental backup logic: Read `/home/user/backup/archive.jsonl`. A record is considered "new" if its `RecordID` does not already exist in the archive.
5. Convert the new, valid records into JSON objects and append them to `/home/user/backup/archive.jsonl`. Ensure the keys in the JSON exactly match the keys in the log (`RecordID`, `Date`, `SensorType`, `Value`), with `Value` stored as a float and the rest as strings. Add a newline after each JSON object.

Do not modify the existing lines in `archive.jsonl`. Just append the new matching records.
