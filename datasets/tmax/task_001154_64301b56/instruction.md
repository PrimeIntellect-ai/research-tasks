You are acting as a security compliance analyst for a small web application. The company has discovered that their internal Python (Flask) web server contains an open redirect vulnerability in its login authentication flow, and they need an audit trail of exploitation attempts, a remediation of the application, and a firewall ruleset to block the attackers.

Your task consists of four parts:

1. **Vulnerability Remediation**:
   The application source code is located at `/home/user/app.py`. It currently blindly redirects users to the URL provided in the `next` query parameter after a simulated login. 
   Modify `/home/user/app.py` so that it only redirects to relative paths. A path is considered safe and relative *only* if it starts with a single forward slash `/` but does NOT start with double slashes `//` (to prevent protocol-relative URL exploits). If the `next` parameter is missing, invalid, or an absolute URL, default the redirect to `/dashboard`.

2. **Content Security Policy (CSP)**:
   Modify the Flask application in `/home/user/app.py` to include a strict Content Security Policy. Append the HTTP header `Content-Security-Policy: default-src 'self'` to all responses generated by the application.

3. **Intrusion Detection & Audit Trail**:
   A web server access log is located at `/home/user/access.log`. You must analyze this log to identify attackers. 
   Find all unique IP addresses that successfully exploited the open redirect. An exploit is defined as a log entry where the HTTP method is `GET`, the path is `/login`, the `next` query parameter contains an absolute URL (starts with `http://` or `https://`), and the HTTP response status code is `302`.
   Write these unique, malicious IP addresses to `/home/user/malicious_ips.txt`, one IP address per line, sorted in ascending order.

4. **Firewall Rule Generation**:
   Generate a shell script at `/home/user/block_rules.sh` that contains iptables commands to block the identified malicious IPs.
   For each IP address in `/home/user/malicious_ips.txt` (in the same sorted order), append a line to `/home/user/block_rules.sh` with the following exact format:
   `iptables -A INPUT -s <IP_ADDRESS> -j DROP`

Ensure all files are saved in the correct locations with the exact names specified.
