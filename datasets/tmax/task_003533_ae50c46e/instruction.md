You are tasked with fixing a broken web application stack deployed in the user's home directory. 

The application consists of an Nginx reverse proxy listening on port 8080 and a Python backend API listening on port 8081. Currently, if you attempt to fetch data from the reverse proxy using `curl http://127.0.0.1:8080/api/report`, it returns a 502 Bad Gateway error. 

There are multiple configuration and environment issues causing this failure:

1. **Nginx Configuration:** The Nginx configuration file is located at `/home/user/nginx/nginx.conf`. It has been misconfigured and is routing traffic to the wrong backend port.
2. **Cron/Job Environment Simulation:** The backend API relies on a JSON report file located precisely at `/home/user/app_data/report.json`. This file is supposed to be generated by a scheduled job script `/home/user/scripts/run_job.sh`, which wraps a Python script `/home/user/scripts/generate_report.py`. However, due to missing environment variables (similar to common crontab environment issues), the script writes to the wrong location and formats the timestamps using the system's default local timezone and locale instead of the required `UTC` timezone and standard `C` locale.
3. **Backend Service:** The Python backend located at `/home/user/backend/server.py` crashes or returns an error if the report file is missing or if the timestamps inside it aren't in the expected UTC format.

Your objectives are to:
1. Fix the Nginx configuration file in `/home/user/nginx/nginx.conf` so it correctly proxies requests for `/api/` to the backend on port 8081.
2. Edit `/home/user/scripts/run_job.sh` to export the environment variable `REPORT_DIR=/home/user/app_data`, set the timezone `TZ` to `UTC`, and set `LC_ALL=C` so that the Python generator script builds the file in the right location with the right timestamp formatting.
3. Execute `/home/user/scripts/run_job.sh` successfully so that `/home/user/app_data/report.json` is generated.
4. Start the backend server by running `python3 /home/user/backend/server.py &`.
5. Start (or reload) Nginx by running `nginx -c /home/user/nginx/nginx.conf -p /home/user/nginx &`.
6. Make a request to the proxy: `curl -s http://127.0.0.1:8080/api/report` and save the exact output to `/home/user/success_response.txt`.

Ensure all processes are started without requiring root privileges (Nginx is configured to run as the local user). If `/home/user/success_response.txt` contains the correct JSON payload, the task is complete.
