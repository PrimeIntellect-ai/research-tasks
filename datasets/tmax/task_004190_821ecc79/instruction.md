You are tasked with building a C++ data processing tool to analyze noisy configuration change logs generated by an enterprise configuration manager. The logs track modifications made to servers, but they suffer from duplicate events, inconsistent target naming, and unstructured details.

Your goal is to write a C++ program (to be saved at `/home/user/tracker.cpp`) that reads `/home/user/config_changes.log`, processes the data according to the rules below, and writes a summarized CSV report to `/home/user/summary.csv`.

### Input Format
The file `/home/user/config_changes.log` contains one log entry per line in the following format:
`[{Timestamp}] User={Username} Action={Action} Target={Target} Details="{Details}"`

Example:
`[2023-11-01T10:05:12Z] User=alice Action=Update Target=/ETC//ssh/sshd_config Details="status: success, lines_added: 12"`

### Processing Rules
1. **Regex Parsing**: Parse each line to extract the Timestamp, User, Action, Target, and Details. Ignore any malformed lines.
2. **Cleaning & Normalization**:
   - Normalize the `Target` field by converting it entirely to lowercase.
   - Replace any consecutive forward slashes (`//`, `///`) in the `Target` with a single slash (`/`).
3. **Deduplication**: The logging system often stutters. If a log entry has the exact same normalized `Target`, `Action`, and `Details` as the *most recently accepted* event for that specific normalized `Target`, it must be ignored as a duplicate. (Note: A new event for Target A does not reset the deduplication state for Target B).
4. **Feature Extraction**: Use regex on the `Details` string to extract the number of lines added. Look for the pattern `lines_added: <number>`. If present, extract the integer. If missing, assume 0 lines added.
5. **Time-Based Bucketing**: Truncate the timestamps to the hour (e.g., `2023-11-01T10:05:12Z` becomes `2023-11-01T10:00:00Z`). Aggregate the accepted (deduplicated) events into these hourly buckets.

### Output Specification
Generate a CSV file at `/home/user/summary.csv` with the following exact headers:
`Hour_Bucket,Total_Changes,Unique_Targets,Total_Lines_Added`

- `Hour_Bucket`: The truncated timestamp string (e.g., `2023-11-01T10:00:00Z`).
- `Total_Changes`: The count of accepted (deduplicated) events in this hour.
- `Unique_Targets`: The number of strictly unique normalized targets modified in this hour.
- `Total_Lines_Added`: The sum of lines added across all accepted events in this hour.

Sort the CSV output chronologically by `Hour_Bucket`.

Please write the code, compile it using `g++ -std=c++17 -O2 /home/user/tracker.cpp -o /home/user/tracker`, and execute it to produce the `summary.csv` file. You may use standard C++ libraries only (e.g., `<regex>`, `<string>`, `<map>`, `<vector>`, `<fstream>`).
