You are a Database Reliability Engineer tasked with auditing a complex database backup and replication topology. Your company uses a specialized backup system that relies on a replication graph to determine the order of backups. However, the backup resolution process has been slow, and the topology has become undocumented.

You have been provided with two files in your home directory (`/home/user`):
1. `replication_graph.json`: Contains the replication topology of your databases. It includes a list of nodes (databases) and directed edges representing replication streams (from `source` to `target`).
2. `backup_query_plan.txt`: Contains an execution plan generated by the backup scheduling engine (similar to a PostgreSQL EXPLAIN plan) that runs a recursive graph traversal to find backup dependencies.

Your task is to analyze these files using a Python script and generate a JSON report. Write a Python script (e.g., `analyze.py`) that performs the following analysis:

1. **Topology Centrality**: Calculate the out-degree (number of outgoing replication streams) for each database. Identify the database name that acts as the primary master (the node with the highest out-degree).
2. **Knowledge Graph Pattern Matching**: High availability requirements dictate that some databases use a "triangular" replication pattern. Find all instances of triangles in the graph. A triangle is defined as three databases (X, Y, Z) where there is a replication stream from X to Y, from X to Z, and from Y to Z. 
3. **Query Plan Interpretation**: Read `backup_query_plan.txt`. The plan lists various operations along with their estimated costs in the format `(cost=startup..total)`. Parse the file to find the specific operation (e.g., "Seq Scan on backup_logs", "Hash Join", "Sort") that has the highest `total` cost value. 

Once you have computed these values, write the results to `/home/user/backup_report.json` with the following strict structure:

```json
{
  "master_db": "name_of_the_database_with_highest_out_degree",
  "triangular_patterns": [
    ["X_name", "Y_name", "Z_name"],
    ...
  ],
  "bottleneck_operation": "Exact name of the operation with the highest total cost"
}
```

Constraints for `backup_report.json`:
- The `triangular_patterns` list must contain lists of EXACTLY three database names `[X, Y, Z]` matching the X->Y, X->Z, Y->Z pattern.
- Each triplet must be sorted internally so that the source node X is first, the intermediate node Y is second, and the target node Z is third.
- Sort the outer list of triplets alphabetically by X_name, then Y_name, then Z_name.
- The `bottleneck_operation` should be the operation name exactly as it appears before the `(cost=...)` block, stripped of leading/trailing whitespace (e.g., "Limit", "Hash Join", "Seq Scan on databases").

Use Python to implement this logic and generate the output file.
