You are a storage administrator managing disk space on a critical Linux server. The server's disk is filling up rapidly due to massive, monolithic Write-Ahead Log (WAL) files generated by a legacy application ecosystem. 

Management has decommissioned a specific internal application that was highly verbose, and we need to reclaim disk space by actively stripping out all its historical log records from our WAL archives.

We received a memo detailing which application was decommissioned, but it was only provided as a scanned image located at `/app/policy.png`. You will need to extract the target "App ID" from this image.

The WAL files use a proprietary binary format. You must write a Python script at `/home/user/compact_wal.py` that reads a WAL stream from standard input (`stdin`) and writes the compacted WAL stream to standard output (`stdout`). 

Because these files can be hundreds of gigabytes in size, you cannot load the entire file into memory at once; you must use streaming I/O, processing and writing the file sequentially.

**WAL Format Specification:**
1. **Global Header**: The first 8 bytes of the file are always the magic string `WALOGv1\0` (in ASCII/hex: `57 41 4c 4f 47 76 31 00`).
2. **Records**: The rest of the file consists of contiguous records. Each record has:
   - `Length`: A 4-byte unsigned integer (little-endian) representing the length of the *Payload* in bytes.
   - `App ID`: A 4-byte unsigned integer (little-endian) representing the application that generated the log.
   - `Payload`: Binary data exactly `Length` bytes long.

**Your Goal:**
Write `/home/user/compact_wal.py` to continuously parse the WAL format from `sys.stdin.buffer`, filtering out (dropping) ANY record where the `App ID` exactly matches the decommissioned App ID extracted from `/app/policy.png`. All other records (and the global header) must be written unaltered to `sys.stdout.buffer`. 

Your script must handle arbitrary payloads without corrupting the file structure, and gracefully exit when the end of the input stream is reached. Make sure the script is executable.

Note: You may install and use OCR tools like `tesseract-ocr` (and its Python bindings) to read the image if necessary, or simply inspect the image text using basic vision extraction.
