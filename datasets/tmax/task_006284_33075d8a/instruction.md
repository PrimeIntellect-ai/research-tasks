You are a compliance analyst tasked with generating audit trails from a legacy, isolated logging mechanism. During a recent security incident, a sandboxed application generated an encrypted log file, but the source code for the logging utility was lost. We suspect the application was targeted by an injection attack.

You have been provided with two files in your home directory:
1. `/home/user/secure_logger.pyc`: The compiled Python bytecode of the legacy logging utility.
2. `/home/user/encrypted_audit.bin`: The encrypted audit log generated by the sandboxed application.

Your task is to:
1. Reverse engineer the `secure_logger.pyc` bytecode (you may use Python's built-in `dis` module or write a script to analyze it) to understand the custom encryption algorithm used. The algorithm is a weak custom linear/XOR cipher.
2. Write a Python script to decrypt the contents of `/home/user/encrypted_audit.bin`.
3. Save the fully decrypted log in plain text to `/home/user/decrypted_audit.txt`.
4. Analyze the decrypted log to identify the specific Injection/XSS vulnerability payload that the attacker attempted to use against the system.
5. Extract *only* the exact malicious payload string and write it to `/home/user/vulnerability_report.txt`.

Ensure your final decrypted log file is exactly as it was originally written before encryption, and the vulnerability report contains only the isolated payload string.
