You are a machine learning engineer preparing synthetic training data for a 3D generative model. You need to implement a fast data generator, evaluate its statistical correctness against a baseline, and run a convergence test.

Your task consists of three parts:

**1. Monte Carlo Data Generator (`/home/user/mc_generator.py`)**
Write a Python script that uses NumPy to implement Monte Carlo rejection sampling. 
- It must sample 3D points $(x, y, z)$ from the unnormalized probability density function:
  $f(x, y, z) = x^2 + y^2 + z^2$
- The domain for sampling is the 3D cube: $x, y, z \in [-1, 1]$.
- The script must take exactly three command-line arguments: `N` (the number of accepted samples to output), `seed` (an integer random seed for reproducibility), and `output_file` (the path to save the generated CSV).
- The output CSV must have a header `x,y,z` and `N` rows of comma-separated floats. Use `numpy.random.default_rng(seed)` for random number generation.

**2. Distribution Metric Evaluator (`/home/user/evaluator.py`)**
Write a Python script to measure the probability distribution distance between two generated datasets.
- It must take two command-line arguments: `file1.csv` and `file2.csv`.
- Read the 3D points from both CSVs using NumPy (multi-dimensional array manipulation).
- For each dataset, compute the Euclidean distance of each point to the origin ($r = \sqrt{x^2+y^2+z^2}$).
- Calculate the 1-Wasserstein distance between the two empirical distributions of radii ($r$) using `scipy.stats.wasserstein_distance`.
- Print only the computed distance to standard output (as a float).

**3. Convergence & Regression Testing (`/home/user/test_convergence.sh`)**
There is a reference dataset of 100,000 points at `/home/user/reference_100000.csv` generated by a slow, verified legacy script.
Write a Bash script that:
- Runs `mc_generator.py` to generate datasets for $N = 100, 1000, 10000$. Use `seed=42` for all three runs. Save them as `gen_100.csv`, `gen_1000.csv`, and `gen_10000.csv`.
- Runs `evaluator.py` to compare each generated dataset against the reference dataset `/home/user/reference_100000.csv`.
- Constructs a JSON file at `/home/user/results.json` mapping the sample size to the computed Wasserstein distance. The JSON must have exactly this structure:
```json
{
  "100": 0.045123,
  "1000": 0.012345,
  "10000": 0.003456
}
```
*(The values above are just examples).*

Ensure all scripts are executable and `test_convergence.sh` correctly populates `/home/user/results.json`.
