You are managing configuration changes across multiple servers. A remote automated system dumps exported configuration change logs as a tarball. Your task is to extract, normalize, and process these logs to identify rejected configuration changes.

Here are the requirements:
1. An archive of the logs is located at `/home/user/remote_stash/configs.tar.gz`. Extract it to `/home/user/raw_configs/`.
2. The archive contains several `.csv` files. Some of these files were generated by legacy Windows systems and are encoded in `UTF-16LE`. You must detect and convert all `.csv` files in the `/home/user/raw_configs/` directory to `UTF-8` encoding.
3. The CSV files have the following header: `server_id,change_id,status,description,timestamp`
4. **Important**: The `description` field is enclosed in double quotes (`"`) and often contains embedded newlines. Standard line-by-line tools like a simple `grep` will break on these multiline records.
5. Extract all records where the `status` column is exactly `REJECTED`.
6. Output a summary of these rejected changes to `/home/user/rejected_changes.txt`. 
7. The output file `/home/user/rejected_changes.txt` must contain exactly one line per rejected change in the format `server_id:change_id`, sorted alphabetically.

You must accomplish this using only standard Linux shell built-ins, coreutils, and standard CLI tools (like `awk`, `iconv`, `grep`, `sed`, `tar`). Do not use Python, Perl, or other higher-level scripting languages.
