As a mobile build engineer, you are maintaining our pipeline's build configuration evaluator. We have a legacy C++ component that parses and evaluates build constraint expressions (e.g., matching target architectures or OS versions). To decouple this from our main orchestrator, we are migrating it into a standalone gRPC microservice.

However, the legacy C++ evaluator has a severe memory safety issue (undefined behavior) that occasionally crashes the pipeline, and it lacks a gRPC wrapper and polyglot build orchestration.

Your task is to fix the memory safety issue, define the gRPC interface, write the C++ gRPC server implementation, orchestrate the multi-language build, and run an automated Python client against it.

### Step 1: Fix the Memory Safety Bug
In `/home/user/workspace/evaluator.cpp`, there is a basic Reverse Polish Notation (RPN) expression evaluator used for build constraints. It takes an expression string and a map of integer variables. 
It contains a memory safety bug (buffer overflow/out-of-bounds access) when handling certain malformed expressions (e.g., popping from an empty stack or pushing too many items). Fix the bug so that `evaluate_rpn` safely returns an error instead of crashing.

### Step 2: Define the gRPC Service
Create a protobuf file at `/home/user/workspace/evaluator.proto` with the following specification:
- **Syntax**: proto3
- **Package**: `build_expr`
- **Service**: `Evaluator`
  - Has an RPC method `Evaluate` that takes an `EvalRequest` and returns an `EvalResponse`.
- **Message `EvalRequest`**:
  - `string expression` (field 1)
  - `map<string, int32> variables` (field 2)
- **Message `EvalResponse`**:
  - `int32 result` (field 1)
  - `bool success` (field 2)
  - `string error_message` (field 3)

### Step 3: Implement the C++ gRPC Server
In `/home/user/workspace/server.cpp`, implement the `build_expr::Evaluator::Service`. We have provided a skeleton. You need to implement the `Evaluate` method to call the `evaluate_rpn` function from `evaluator.h`. 
Bind the server to `0.0.0.0:50051`.

### Step 4: Polyglot Build Orchestration
Create a `Makefile` in `/home/user/workspace` that handles building both the C++ gRPC server and generating the Python gRPC stubs.
The `Makefile` must have the following targets:
- `all`: Builds the C++ server (binary named `evaluator_server`) and generates the Python stubs.
- `evaluator_server`: Compiles the C++ code, linking against `protobuf`, `grpc`, and `grpc++`. You must use `protoc` to generate the C++ pb and grpc.pb files from `evaluator.proto`.
- `python_stubs`: Uses the `grpc_tools.protoc` module to generate `evaluator_pb2.py` and `evaluator_pb2_grpc.py` in the workspace.

*Note: Assume `grpc`, `grpc++`, and `protobuf` are installed in standard system paths, and `python3 -m grpc_tools.protoc` is available.*

### Step 5: Test Execution
Once built, start your `evaluator_server` in the background.
We have provided a Python script at `/home/user/workspace/client.py` which will read expressions from `/home/user/workspace/test_cases.json`, call your gRPC server, and write the output.
Run `python3 client.py /home/user/workspace/test_cases.json /home/user/workspace/results.log`.

The final output must be precisely written to `/home/user/workspace/results.log` in the format generated by `client.py`.
