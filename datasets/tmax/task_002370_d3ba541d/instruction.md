You are an AI assistant helping a research scientist organize and analyze a dataset of academic paper citations. 

The researcher has two raw CSV files in `/home/user/`:
1. `/home/user/nodes.csv` - Contains paper metadata with columns `id,title,year`.
2. `/home/user/edges.csv` - Contains the citation graph with columns `source,target`. A row where source is "A" and target is "B" means "Paper A cites Paper B".

Your task is to create a robust Bash script at `/home/user/process_citations.sh` that automates the ingestion, graph traversal, and extraction of this data using `sqlite3`. 

The bash script must perform the following pipeline:
1. **Materialization**: Create a new SQLite database at `/home/user/citations.db`. Create tables `nodes` and `edges` and import the data from the respective CSV files. Make sure the CSV headers are handled correctly.
2. **Graph Traversal & Joins**: Execute a query that traverses the citation graph to find all papers cited by the paper with `id` exactly equal to `P001`, up to a maximum depth of 3. (Depth 0 is `P001` itself, Depth 1 are papers directly cited by `P001`, Depth 2 are papers cited by Depth 1 papers, etc.). Ensure you join this traversal with the `nodes` table to retrieve the titles of these papers.
3. **Export and Formatting**: The query result must be exported as a strictly formatted JSON array to `/home/user/citation_tree.json`. Each object in the JSON array must have exactly the following keys:
   - `depth` (integer)
   - `paper_id` (string)
   - `title` (string)
   The JSON array must be ordered by `depth` ascending, and then by `paper_id` ascending.

Constraints & Requirements:
- Write the entire workflow in standard Bash (`/home/user/process_citations.sh`). Make it executable.
- Use `sqlite3` for the database and querying operations. You may assume `sqlite3` is installed. Use SQLite's recursive Common Table Expressions (CTEs) for the graph traversal.
- The script should be self-contained and assume the CSV files already exist.
- Do not hardcode the expected JSON output in the script; it must be dynamically generated by querying the database.

When you are finished, ensure `/home/user/process_citations.sh` has `+x` permissions and is ready to be run by the testing environment.
