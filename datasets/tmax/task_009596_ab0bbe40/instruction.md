You are a data engineer responsible for processing legacy server logs into a clean, queryable format for downstream analytics.

You have been given a directory of raw log files located at `/home/user/raw_logs/`. These files are generated by different legacy systems, which unfortunately means they use different character encodings (specifically, some use `UTF-16LE` and others use `ISO-8859-1`).

Your task is to write and execute a Python script (save it as `/home/user/etl.py`) that performs an ETL pipeline with the following requirements:

1. **Extraction & Encoding Handling**:
   - Read all `.log` files in `/home/user/raw_logs/`.
   - Gracefully handle the different character encodings (you may attempt to decode using `utf-16le` and `iso-8859-1`, falling back as necessary).
   - Each valid log line follows this exact format:
     `[YYYY-MM-DD HH:MM:SS] IP:<ip_address> CPU:<load>% MEM:<mem_mb>MB STATUS:<status>`
   - Extract the timestamp, IP address, CPU load (as an integer), and Memory (as an integer).

2. **Validation**:
   - Discard any lines that do not strictly match the expected format.
   - Discard records where the CPU load is less than 0 or greater than 100.
   - Discard records where the IP address is not a valid IPv4 address.

3. **Time-Based Bucketing & Aggregation**:
   - Group the valid data into 1-hour time buckets based on the timestamp (e.g., records from `14:15:30` and `14:45:00` both fall into the `14:00:00` bucket).
   - For each 1-hour bucket and each unique IP address, calculate:
     - The **average CPU load** (as a float).
     - The **maximum Memory usage** (as an integer).

4. **Database Export**:
   - Create an SQLite database at `/home/user/metrics.db`.
   - Create a table named `hourly_metrics` with the following schema:
     - `bucket_hour` (TEXT, format: 'YYYY-MM-DD HH:00:00')
     - `ip_address` (TEXT)
     - `avg_cpu` (REAL)
     - `max_mem` (INTEGER)
   - Insert the aggregated results into this table.

Run your script to populate `/home/user/metrics.db` so the automated systems can verify your pipeline's output.
