You are tasked with replacing a legacy configuration management system. We have an old, undocumented utility that packs and unpacks configuration directories into a proprietary archive format (`.ccf`). 

Recently, a security audit revealed that the legacy extractor is vulnerable to "Zip Slip" (path traversal) attacks when unpacking malicious `.ccf` files. While the security team is writing a safe extractor, your task is to write a clean, 100% compatible Python implementation of the *packer* so we can begin decommissioning the old binary from our build pipelines.

You have been provided with the legacy stripped binary at:
`/app/legacy_cfg_manager`

It supports two commands:
1. `/app/legacy_cfg_manager pack <input_dir> <output.ccf>`
2. `/app/legacy_cfg_manager unpack <archive.ccf> <output_dir>`

Your Objective:
1. Reverse engineer the `.ccf` (Custom Config Format) archive structure by interacting with the `/app/legacy_cfg_manager` binary. You can use standard Linux tools (e.g., `xxd`, `strace`, `ltrace`, python, etc.) to analyze its inputs and outputs.
2. Write a Python script at `/home/user/pack_ccf.py` that perfectly replicates the `pack` functionality.
3. Your script must accept exactly two arguments:
   `python3 /home/user/pack_ccf.py <input_directory> <output_file.ccf>`
4. The script must recursively read all files in `<input_directory>` and generate a `.ccf` archive.
5. **Crucial Requirement:** The archive generated by your Python script must be **bit-for-bit identical** to the archive generated by the legacy binary for the exact same directory contents. Note that the legacy binary always sorts files in a specific order to ensure deterministic builds.

Ensure your code handles various file sizes and directory structures (nested directories, binary files, text files). You do not need to implement the `unpack` functionality.
