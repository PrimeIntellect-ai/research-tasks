You are a security automation engineer. We have a simple bash-based validation utility that acts as a mock Web Application Firewall (WAF) filter, checking input payloads against a denylist configuration file. We suspect the filter is vulnerable to case-variation bypasses.

Your task is to write and execute a bash testing script that generates test fixtures, orchestrates the testing against the validator, and patches the configuration file to fix any discovered bypasses.

Here is the setup:
1. The validator script is located at `/home/user/validator.sh`. It reads a single payload string from standard input (stdin). It exits with status code `1` if the payload is blocked, and `0` if it is allowed.
2. The denylist configuration file is located at `/home/user/config.txt`.
3. We need to test the specific target payload: `admin`.

You must perform the following actions:
1. **Property-based Test Generation**: Generate all possible uppercase and lowercase permutations of the word `admin` (e.g., `admin`, `Admin`, `aDmin`, `ADMIN`, etc.).
2. **End-to-end Test Orchestration**: Feed each permutation into `/home/user/validator.sh` via standard input.
3. **Capture Bypasses**: Identify which permutations successfully bypass the filter (i.e., cause the validator to exit with `0`). Write exactly these bypassing permutations to `/home/user/bypasses.log`, one per line, sorted alphabetically.
4. **Patch Generation**: 
   - Create a backup of the original config: `cp /home/user/config.txt /home/user/config.txt.orig`
   - Append all discovered bypassing permutations to `/home/user/config.txt` so they are now blocked.
   - Use the `diff` command to generate a unified patch file showing the changes. Save this patch to `/home/user/config.patch`. The patch should be generated by comparing `/home/user/config.txt.orig` to `/home/user/config.txt`.

Ensure your final state includes the correctly populated `/home/user/bypasses.log`, the updated `/home/user/config.txt`, and the unified diff in `/home/user/config.patch`.
