You are an AI assistant helping a robotics researcher organize a large dataset of experimental runs. 

The researcher has a dataset located at `/home/user/dataset`. Inside this directory, there are multiple subdirectories representing different experimental runs (e.g., `run_01`, `run_02`). Each run directory contains a mix of files generated by the robotic system:
- `.gcode` files: The movement commands sent to the robotic arm.
- `.elf` files: The compiled firmware binaries used for that specific run.
- `.wal` files: SQLite Write-Ahead Logs containing sensor telemetry.

Your task is to write and execute a Go program (`/home/user/analyzer.go`) that recursively navigates the `/home/user/dataset` directory, analyzes these specific file types concurrently, and generates a unified, cryptographically hashed manifest.

The Go program must meet the following requirements:
1. **Concurrency & Locking:** It must process the files concurrently using goroutines. Since multiple goroutines will be extracting data and need to write to a single output file `/home/user/manifest.jsonl`, you must use strict file locking (e.g., `syscall.Flock` with `LOCK_EX`) whenever writing to the manifest to prevent interleaving or corruption.
2. **Checksum Generation:** Compute the SHA-256 hash (hex-encoded string) for every `.gcode`, `.elf`, and `.wal` file.
3. **Domain-Specific Parsing:**
   - **GCode (`.gcode`)**: Parse the text file and find the maximum Z-axis height reached. Look for lines starting with `G1` or `G0` that contain a `Z` parameter (e.g., `G1 X10 Y10 Z25.5`). Extract the numeric value. The metadata extracted should be `{"max_z": 25.5}`.
   - **ELF (`.elf`)**: Parse the ELF binary header to extract the Entry Point address (as a decimal integer). The metadata should be `{"entry_point": 4194304}`.
   - **WAL (`.wal`)**: SQLite WAL files have a 32-byte file header. The 32-bit big-endian integer at offset 8 is the database page size. A WAL frame consists of a 24-byte frame header followed by `page_size` bytes of data. Calculate the total number of frames in the WAL file: `(file_size - 32) / (24 + page_size)`. The metadata should be `{"frames": 12}`.
4. **Output Format:** For each processed file, append a single JSON object on a new line (JSONL format) to `/home/user/manifest.jsonl`. 
   Format: `{"filepath": "/home/user/dataset/run_01/arm.gcode", "type": "gcode", "sha256": "...", "meta": <the extracted metadata object>}`

Write the Go program, compile it, and run it against the `/home/user/dataset` directory. Leave the resulting `/home/user/manifest.jsonl` file on the disk for verification.
