As an incident responder, you have been called to investigate a suspicious breach on one of our web servers. The attacker managed to steal SSH private keys and bypass our Content Security Policy (CSP) enforcement by tunneling the stolen data through fake CSP violation reports.

During the investigation, we recovered the attacker's exfiltration program: a stripped executable located at `/app/exfil_tool`. This binary reads sensitive files, applies a custom obfuscation algorithm, and formats the output to look like a CSP violation report. 

We have collected a large number of these CSP log files. Some are completely legitimate CSP violation reports generated by real browsers. Others are malicious reports containing obfuscated SSH keys exfiltrated by the attacker.

Your task:
1. **Reverse Engineer the Obfuscator:** Analyze `/app/exfil_tool` (using tools like `objdump`, `strings`, `gdb`, etc.) to understand the custom cryptography/obfuscation used by the attacker to hide the sensitive data.
2. **Develop a Detector:** Write a Python script at `/home/user/detector.py` that takes a single file path as a command-line argument.
3. **Analyze and Classify:** The script must read the provided JSON CSP log file. If the file contains a malicious exfiltration payload that successfully decrypts/deobfuscates to reveal an SSH private key (containing the string `PRIVATE KEY`), the script must print `EVIL` to standard output and exit with status code `1`. If the file is a legitimate CSP report (even if it contains suspicious fields that do NOT decode to an SSH key), it must print `CLEAN` to standard output and exit with status code `0`.

**Constraints & Notes:**
- The malicious payload is embedded within the `"violated-directive"` field of the CSP report JSON in the format `exfil-<encoded_payload>`.
- Be careful: Some clean logs may contain fake `exfil-` directives added by the attacker as a decoy, which decode to harmless text. Your script must actually deobfuscate the payload and check for the sensitive data signature to avoid false positives.
- You must use Python 3 for your script.
- The automated grading system will run your script against a hidden adversarial corpus of hundreds of logs. You must achieve 100% accuracy to pass.
