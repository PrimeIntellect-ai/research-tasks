You are a backup administrator tasked with writing a secure extraction tool in Rust for a legacy Write-Ahead Log (WAL) archive format. Recently, an archive was discovered to have been generated by a compromised system that injected "zip slip" style directory traversal paths into the archive. 

Your objective is to write a Rust command-line tool that recursively finds these WAL files, safely parses and extracts them, and logs any malicious paths that attempt to escape the designated extraction directory.

System State & Requirements:
1. There is a configuration file at `/home/user/backup_config.toml` containing:
   ```toml
   source_dir = "/home/user/archives"
   target_dir = "/home/user/extracted"
   ```
2. The `/home/user/archives` directory contains various files, including custom `.wal` archives nested within subdirectories.
3. The `.wal` format is a custom binary format. It consists of sequential entries with no global header or footer. Each entry contains:
   - `path_len`: 16-bit unsigned integer (Little Endian)
   - `path`: UTF-8 string of length `path_len`
   - `data_len`: 32-bit unsigned integer (Little Endian)
   - `data`: Raw bytes of length `data_len`
4. Create a Rust project at `/home/user/wal_extractor`. Your tool must:
   - Read the paths from `/home/user/backup_config.toml`.
   - Recursively traverse the `source_dir` to find all files ending in `.wal`.
   - Stream through each `.wal` file (avoiding loading the entire archive into memory at once).
   - Compute the absolute normalized path of each file entry as if it were extracted inside `target_dir`.
   - **Security Check:** If the normalized absolute path does not start with the exact path of `target_dir` (i.e., it attempts a directory traversal outside `/home/user/extracted`), it is a slip attempt. 
   - Write any malicious/escaping `path` string exactly as it appears in the WAL file to `/home/user/slip_attempts.log` (one path per line, in the order they are encountered). Do NOT extract this file.
   - For safe paths, ensure the necessary parent directories exist within `target_dir` and write the data payload to the target file.
   
5. Compile and run your tool to process the archives. 

Verify your success by ensuring the valid files are extracted in `/home/user/extracted` and `/home/user/slip_attempts.log` contains the paths that failed the security check.
