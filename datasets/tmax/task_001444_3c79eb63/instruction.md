You are a log analyst investigating server performance patterns. The application team has provided a raw log file `/home/user/raw_logs.txt` generated by a legacy system. The file contains millions of events, but they are messy and use an obfuscated format.

Your task is to build a fast data processing pipeline using **C** and standard bash utilities to extract, validate, normalize, and summarize this data.

### Input Data Format
The file `/home/user/raw_logs.txt` is a pipe-delimited (`|`) text file.
Columns: `timestamp|ip_address_hex|user_id|action|response_time_ms|status_code`
Example:
`1700000000|0x7F000001|U100|GET|150|200`

### Requirements

**Phase 1: The C Processor**
Write a C program at `/home/user/log_processor.c` and compile it to `/home/user/log_processor`.
The program must read the pipe-delimited logs from `stdin` and output cleaned, normalized CSV records to `stdout`.
It must apply the following logic:
1. **Validation (Quality Gate):** 
   - Drop the entire row if `status_code` is not an integer between `200` and `599` (inclusive).
   - Drop the entire row if `response_time_ms` is strictly less than `0`.
   - Drop the row if it lacks exactly 6 columns.
2. **Feature Extraction & Normalization:**
   - Convert `timestamp` (Unix epoch) to an integer `hour_id` (calculated as `timestamp / 3600`).
   - Convert `ip_address_hex` from its hexadecimal representation (e.g., `0x7F000001`) to standard dotted-decimal notation (e.g., `127.0.0.1`).
3. **Output Format:**
   - Print valid records to stdout as a comma-separated list: `hour_id,ip_address,user_id,response_time_ms`

**Phase 2: Aggregation and Sorting**
Using your compiled C program and standard bash tools (`sort`, `awk`, etc.), process `/home/user/raw_logs.txt` and generate a final summary statistics file at `/home/user/hourly_stats.csv`.

The final file `/home/user/hourly_stats.csv` must:
1. Be grouped by `hour_id` and then `user_id`.
2. Contain the following comma-separated columns:
   `hour_id,user_id,total_requests,avg_response_time_ms`
3. The `avg_response_time_ms` must be the arithmetic mean of the response times for that group, rounded down to the nearest integer (floor).
4. The file must be sorted by `hour_id` (numeric, ascending), then by `user_id` (alphanumeric, ascending).
5. Do not include a header row.

Constraints:
- You must write and use the C program for Phase 1.
- Ensure the final output file exists exactly at `/home/user/hourly_stats.csv`.
