You are a network engineer troubleshooting a custom CI/CD health-check pipeline. 

Your team has a background watcher service that periodically executes a health probe script to monitor local network connectivity and storage usage. However, the probe is currently failing to record data correctly. The watcher executes the script with a stripped environment (simulating a strict CI runner), causing the script to fail due to `PATH` issues and relative directory misplacements (writing to a random working directory instead of the designated logs folder).

Your task is to fix the health monitor script, ensure network connectivity is properly logged, and implement a storage threshold alert.

**System Details and Requirements:**
1. A local python web service runs at `http://localhost:8080/ping`. (You must start this service in the background: `python3 /home/user/app.py &`).
2. The background watcher script is located at `/home/user/watcher.py`. (You must start this in the background: `python3 /home/user/watcher.py &`). It executes `/home/user/health_monitor.sh` every 2 seconds.
3. Fix `/home/user/health_monitor.sh` so that it overcomes the stripped environment (e.g., by setting explicit paths or exporting a valid `PATH`).
4. The script must ping `http://localhost:8080/ping` using `curl` and append the HTTP status code (e.g., `200`) on a new line to exactly `/home/user/health_logs/status.log`. Do not use relative paths for logging.
5. **Storage Monitoring:** Add logic to `/home/user/health_monitor.sh` to check the file size of `/home/user/health_logs/status.log` in bytes. If the size strictly exceeds 100 bytes, the script must append the exact string `STORAGE_ALERT` on a new line to `/home/user/health_logs/alerts.log`.

**Completion Criteria:**
- The `/home/user/health_logs/status.log` file must exist, be in the correct location, and contain valid HTTP status codes generated by the running watcher.
- The `/home/user/health_logs/alerts.log` file must be generated once the status log exceeds 100 bytes, containing the `STORAGE_ALERT` string.
- The `health_monitor.sh` script must remain executable and be continuously runnable by `watcher.py`.
