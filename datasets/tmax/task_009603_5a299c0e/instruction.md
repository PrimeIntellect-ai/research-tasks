You are a data engineer tasked with building a reproducible ETL pipeline using Rust and Bash.

Your goal is to build a Rust CLI application that computes text "embeddings" (simple deterministic feature extraction), applies a regression model, and performs similarity retrieval. You will also write a Bash script to orchestrate the pipeline reproducibly.

**Step 1: The Rust Application**
Create a new Rust project at `/home/user/etl_tool`.
The application must accept three subcommands: `embed`, `predict`, and `retrieve`.

*   **Subcommand 1:** `cargo run --release -- embed <input_txt> <output_csv>`
    *   Reads a text file line-by-line.
    *   Computes a 3-dimensional "embedding" for each line consisting of:
        1.  `v`: Vowel count (A, E, I, O, U - case-insensitive)
        2.  `c`: Consonant count (ASCII letters that are not vowels - case-insensitive)
        3.  `d`: Digit count ('0' through '9')
    *   Writes a CSV to `<output_csv>` with a header `id,v,c,d`.
    *   The `id` is the 0-indexed line number.

*   **Subcommand 2:** `cargo run --release -- predict <input_csv> <output_txt>`
    *   Reads the CSV generated by `embed`.
    *   Computes a regression score for each row using the formula: `score = (v * 1.5) + (c * -0.5) + (d * 2.0)`
    *   Writes a text file to `<output_txt>` containing `id,score` format for each line. Format the score to exactly one decimal place (e.g., `12.5`). Do not include a header.

*   **Subcommand 3:** `cargo run --release -- retrieve <input_csv> <v> <c> <d>`
    *   Reads the CSV generated by `embed`.
    *   Finds the row that is closest to the query `[v, c, d]` using squared Euclidean distance: `(v1-v2)^2 + (c1-c2)^2 + (d1-d2)^2`.
    *   If there is a tie, pick the lowest `id`.
    *   Prints *only* the matching `id` to standard output.

**Step 2: The ETL Pipeline Script**
Create a Bash script at `/home/user/pipeline.sh`.
This script must be executable (`chmod +x`) and perform the following exact sequence when run:
1. Navigate to `/home/user/etl_tool` and compile the Rust project in release mode.
2. Run the `embed` command using `/home/user/data/raw.txt` as input, outputting to `/home/user/data/embeddings.csv`.
3. Run the `predict` command using the generated CSV, outputting to `/home/user/data/predictions.txt`.
4. Run the `retrieve` command using the generated CSV and the query `v=5`, `c=10`, `d=3`. Redirect the standard output to `/home/user/data/closest.txt`.

*Note: You may use standard crates like `csv` if you update the `Cargo.toml`, but standard library only is also perfectly fine.*
