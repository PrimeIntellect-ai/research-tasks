You are tasked with setting up a polyglot build and test system from scratch. Your goal is to write a C++ program, a Python script, and a Makefile that work together in a data processing pipeline.

All your work must be done inside the `/home/user/polyglot_build` directory.

Here are the requirements for the system:

**1. C++ Parser (`parser.cpp`)**
Write a C++ program that reads comma-separated values (CSV) from standard input (`stdin`). 
Each line of the input will be in the format: `id,name,value` (e.g., `10,widget,50`).
The program must serialize this data into a JSON array of objects and print it to standard output (`stdout`). 
The JSON objects must have the following keys:
*   `"id"`: integer
*   `"name"`: string
*   `"value"`: integer
*   `"arch"`: string

The `"arch"` value must be determined at compile-time using conditional compilation macros:
*   If compiled with `-DTARGET_X86`, the `"arch"` value should be `"x86"`.
*   If compiled with `-DTARGET_ARM`, the `"arch"` value should be `"arm"`.

**2. Python Processor (`processor.py`)**
Write a Python 3 script that reads the JSON array generated by the C++ program from standard input (`stdin`).
The script must perform the following operations:
1.  **Sort** the list of objects by the `"id"` field in ascending order.
2.  **Merge** objects with the same `"id"`. When merging, sum their `"value"` fields together. Keep the `"name"` and `"arch"` from the first object encountered in the sorted order.
3.  **Serialize** the resulting list back into a formatted JSON string (using `json.dumps(..., indent=2)`) and print it to standard output.

**3. Build and Test System (`Makefile`)**
Write a `Makefile` with the following targets:
*   `all`: Compiles `parser.cpp` into two executables: `parser_x86` (using `-DTARGET_X86`) and `parser_arm` (using `-DTARGET_ARM`). Use `g++`.
*   `test`: Runs the full pipeline for both architectures using the input file `/home/user/polyglot_build/input.txt` (you should create a sample one for testing, but your code must work for any valid input). 
    *   It should pipe the input into `parser_x86`, pipe that output into `processor.py`, and redirect the final output to `/home/user/polyglot_build/output_x86.json`.
    *   It should do the same for `parser_arm` and output to `/home/user/polyglot_build/output_arm.json`.

**Initial Setup Steps You Must Take:**
1.  Create the directory `/home/user/polyglot_build`.
2.  Create a sample file `/home/user/polyglot_build/input.txt` with the following exact contents to test your pipeline:
```
3,apple,10
1,banana,20
3,apple,15
2,cherry,5
1,banana,5
```
3. Write `parser.cpp`, `processor.py`, and `Makefile`.
4. Run `make all` and then `make test`.

Verification will check the exact contents of `/home/user/polyglot_build/output_x86.json` and `/home/user/polyglot_build/output_arm.json` after running your Makefile against the specified `input.txt`.
