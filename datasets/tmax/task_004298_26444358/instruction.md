There is a scanned policy document located at `/app/policy.png`. You must use OCR (e.g., `tesseract`) to read this image. It contains the specific string signature that identifies a runaway "log bomb" stack trace. 

**Step 2: Build the Classifier/Sanitizer**
Write a Python script at `/home/user/filter_logs.py` that takes two arguments: an input directory (containing `.tar.gz` log archives) and an output directory.
- The script must extract and inspect the contents of each `.tar.gz` file in the input directory.
- It must parse the multi-line log records inside.
- If a log contains the exact "log bomb" string signature extracted from the policy image, the archive is classified as "evil" and MUST NOT be copied to the output directory.
- If the log does not contain the signature, it is "clean" and the `.tar.gz` MUST be copied unchanged to the output directory.

You must run your script against both corpora and output the surviving files to `/home/user/processed_logs/`. 
Finally, generate a SHA256 checksum manifest of all surviving `.tar.gz` files in `/home/user/processed_logs/` and save it to `/home/user/retention_manifest.txt`. Format: `<sha256>  <filename>`.

Ensure your script is executable and strictly adheres to the classification rules.
