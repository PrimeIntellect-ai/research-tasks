You are acting as an AI assistant helping a data researcher organize and analyze a large academic collaboration dataset. 

The researcher has an edge list of academic co-authorships in `/home/user/dataset/coauthors.csv` (columns: `author_A`, `author_B`, `collaborations`). 

Previously, the lab used a proprietary compiled tool located at `/app/node_scorer` to calculate a "Research Impact Score" for every author. However, this binary is extremely slow on large datasets, and the original source code has been lost. The researcher knows that the algorithm inside the binary calculates a simple linear combination of two standard graph metrics:
1. PageRank (using standard parameters, alpha=0.85)
2. Local Clustering Coefficient

Your task is to:
1. Reverse-engineer the exact linear weights used by `/app/node_scorer` by testing it on small, synthetic graphs. (You can generate dummy CSV edge lists and pass them to the binary like this: `/app/node_scorer dummy.csv`).
2. Write a highly optimized Python script `/home/user/fast_ranker.py` that processes `/home/user/dataset/coauthors.csv` using the `networkx` library to calculate the exact same "Research Impact Score" for all nodes.
3. Your script must sort the authors by their computed score in descending order, and implement pagination. 
4. The script must output the first page of results to `/home/user/top_authors_page1.json`. The JSON must strictly follow this schema:
```json
{
  "page": 1,
  "page_size": 50,
  "total_nodes": <integer>,
  "results": [
    {
      "node_id": "<author_name>",
      "score": <float>
    }
  ]
}
```

Write the Python script, execute it, and generate the JSON file. Our automated test suite will read your JSON file and compute the Mean Squared Error (MSE) between your scores and the true scores generated by the original algorithm on the full dataset.
