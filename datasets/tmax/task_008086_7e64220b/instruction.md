You are a monitoring specialist for a company. Your team stores critical filesystem configuration and backup strategy scripts in a local Git server. You need to set up a monitoring mechanism that generates automated alerts whenever these critical files are modified and pushed to the repository.

Perform the following tasks:

1. **Idempotent Mount Configuration Script**:
   Write a bash script at `/home/user/setup_config.sh`. This script must be strictly idempotent. When run, it should:
   - Ensure the directory `/home/user/backup_mnt` exists.
   - Ensure the file `/home/user/local_fstab` exists.
   - Ensure `/home/user/local_fstab` contains the exact line: 
     `/dev/disk/by-label/BACKUP /home/user/backup_mnt ext4 rw,nosuid,nodev,noexec,auto,nouser,async 0 2`
   - It must *not* add duplicate lines if the line already exists.

2. **Git Server Configuration**:
   Initialize a bare Git repository at `/home/user/config.git`.

3. **Go-based Monitoring Hook**:
   Write a Go program at `/home/user/hook.go`. Compile this program and install it as an executable Git hook at `/home/user/config.git/hooks/post-receive`.
   - The hook must read lines from standard input in the standard `post-receive` format: `<oldrev> <newrev> <refname>`.
   - For each line, the hook must determine which files were modified in the new revision (you can invoke `git diff-tree --no-commit-id --name-only -r <newrev>` within the Go code).
   - If the modified files include `local_fstab` or `backup.sh`, the hook must append a JSON record to `/home/user/alerts.log` for *each* matching file.
   - The JSON record must be exactly in this format on a single line:
     `{"event": "alert", "reason": "critical_config_change", "file": "<filename>", "revision": "<newrev>"}`

4. **Implementation and Testing**:
   - Run `/home/user/setup_config.sh` to generate the initial `local_fstab`.
   - Clone your bare repository to `/home/user/workspace`.
   - Inside `/home/user/workspace`, copy the generated `/home/user/local_fstab` into the repository root.
   - Create a file named `backup.sh` in the repository root with the content `echo "Backup started"`.
   - Commit these two files with the message "Initial config" and push to the `master` branch.

Ensure that after your push, `/home/user/alerts.log` contains the two correct JSON alert lines generated by your Go hook.
