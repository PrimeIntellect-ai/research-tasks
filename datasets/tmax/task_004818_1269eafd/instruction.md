You are an artifact manager in charge of curating a local binary repository. We recently migrated our backend from a legacy Windows environment to a Linux-based system. 

Your task is to write and execute a Bash script at `/home/user/curate_repo.sh` that updates the metadata files for our artifacts. 

The repository is located at `/home/user/artifact_repo/`. Inside this directory (and its subdirectories), there are several metadata files ending in `.meta`. 

Currently, these `.meta` files suffer from two issues:
1. They are encoded in `UTF-16LE` because they were generated by the old Windows system.
2. They contain hardcoded legacy paths that need to be updated.

We have a configuration file located at `/home/user/repo_config.ini` that dictates the migration paths. It looks like this:
```ini
[Paths]
OldPrefix=C:\OldSystem\Binaries\
NewPrefix=/opt/new_system/artifacts/
```

Your script `/home/user/curate_repo.sh` must perform the following actions:
1. **Configuration Interpretation**: Dynamically read `/home/user/repo_config.ini` to extract the `OldPrefix` and `NewPrefix` values. (Do not hardcode these values in your script, as the config file might change in other environments).
2. **Character Encoding Conversion**: Convert every `.meta` file in `/home/user/artifact_repo/` from `UTF-16LE` to `UTF-8`.
3. **Text Editing**: Replace all occurrences of the `OldPrefix` with the `NewPrefix` in the converted text. Be careful with escaping Windows backslashes in your text manipulation.
4. **Atomic Writes**: To prevent repository corruption in case of a crash, you must strictly use atomic writes. For each `.meta` file, write your processed UTF-8 text to a temporary file named `<original_filename>.tmp` in the same directory, and then use `mv` to overwrite the original `.meta` file.

Once you have written the script, execute it to process the repository. 

**Acceptance Criteria**:
- The script `/home/user/curate_repo.sh` exists, is executable, and dynamically parses the `.ini` file.
- All `.meta` files are encoded in UTF-8.
- All `.meta` files have the old Windows paths replaced with the new Linux paths.
- The original files were replaced atomically (this will be verified by checking your script's logic).
