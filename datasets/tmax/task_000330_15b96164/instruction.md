2. The VM operates on a set of 16 string registers (indexed 0 to 15), initially empty.
3. The VM reads binary bytecode from a file provided as the first command-line argument.
4. Each instruction is 4 bytes:
   - Byte 0: Opcode
   - Byte 1: Register A (Destination/Operand)
   - Byte 2: Register B (Operand)
   - Byte 3: Register C (Operand)
5. Known opcodes:
   - `0x01`: SET. Reads a null-terminated string immediately following the instruction in the bytecode and stores it in Register A. (Instruction length varies).

6. The missing opcodes (`0x10`, `0x11`, `0x12`, `0x13`) are described in the audio file. They involve operations like concatenating, sorting characters, reversing, and diffing.
7. Your final program must be executable. Create a shell script at `/home/user/run_vm.sh` that takes the bytecode file path as its first argument, executes your VM, and outputs the result to standard out.
