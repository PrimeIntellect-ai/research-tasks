You are helping a researcher organize and sanitize a massive dataset of academic collaboration networks. The system involves several interacting services, and you must reconfigure them to work properly together, then write a classifier to filter out artificial "fake collaboration rings" generated by bots.

Your tasks are as follows:

1. **Service Configuration & Integration**
   In `/app/services/`, there is a multi-service architecture running:
   - **Redis** (caching and message queue)
   - **Flask API** (port 5000, receives graph datasets)
   - **Worker** (Python background process that calculates graph analytics)
   
   Currently, the Flask API and Worker are failing to communicate. You need to fix their configuration files in `/app/services/api/config.py` and `/app/services/worker/settings.py` so that the API successfully sends tasks to the Worker via Redis, and the Worker correctly processes them. Start the services using `/app/services/start_all.sh`. Once working, sending a POST request with a JSON graph to `http://localhost:5000/process` should return a job ID, and eventually the analytics results.

2. **Graph Analytics Sanitizer**
   The researcher has discovered that automated bots have been inserting fake collaboration networks into the dataset. These fake networks (graphs) exhibit a specific topological anomaly:
   - There is at least one "hub" node with a degree of 5 or more.
   - The local clustering coefficient of this hub node is exactly 0.0 (i.e., none of its neighbors are connected to each other).

   Write a CLI script at `/home/user/sanitizer.py` that takes a path to a JSON file containing a graph's edge list as its first argument. The script should read the JSON file, compute the necessary graph analytics (you may use `networkx`), and:
   - Exit with status code `1` (Reject) if the graph matches the fake collaboration profile (i.e., it contains at least one such anomalous hub).
   - Exit with status code `0` (Accept) if the graph is clean.

   The JSON files are formatted as a list of edge dictionaries, e.g.:
   `[{"source": "NodeA", "target": "NodeB"}, {"source": "NodeB", "target": "NodeC"}]`
   Assume the graphs are undirected.

You have access to a clean corpus and an evil corpus for testing your script:
- Clean corpus: `/home/user/corpus/clean/`
- Evil corpus: `/home/user/corpus/evil/`

Ensure your `sanitizer.py` correctly classifies all files in both directories.
