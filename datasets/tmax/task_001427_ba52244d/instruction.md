You are tasked with organizing and securing a set of legacy project archives for our development team. 

The archives use a custom, uncompressed binary format called "Project Zip Archive" (`.pza`). Recently, we discovered that some archives were generated by a compromised third-party tool and contain "Zip Slip" vulnerabilities—malicious file paths designed to overwrite sensitive files outside the intended extraction directory.

Your objective is to write a Rust command-line utility that safely extracts `.pza` files, completely rejecting any malicious archives, and bulk-renames the extracted files according to a confidential project prefix.

### 1. The Prefix (Image Fixture)
There is an image file located at `/app/project_spec.png`. It contains a scanned label with our project metadata. Use OCR (e.g., `tesseract`) to read the image. Find the string immediately following the text `PREFIX: `. This string (e.g., `ALPHA_123`) must be prepended to the base name of every file you extract (e.g., extracting `src/main.c` becomes `src/ALPHA_123_main.c`).

### 2. The `.pza` Format
You must implement a parser for the `.pza` format in Rust. We recommend using streaming or memory-mapped I/O for efficiency. 
The binary structure is strictly little-endian:
- **Magic Bytes (4 bytes):** `PZA\x01`
- **File Count (4 bytes, `u32`):** Number of files in the archive.
- **File Entries (repeated for File Count):**
  - **Path Length (2 bytes, `u16`):** Length of the file path string in bytes.
  - **Path (variable length):** UTF-8 encoded file path (e.g., `docs/readme.txt`).
  - **Data Length (4 bytes, `u32`):** Size of the file payload in bytes.
  - **Data (variable length):** The raw file bytes.

### 3. Zip Slip Prevention (Adversarial Validation)
Your extractor must meticulously validate every path in the archive *before* writing any files to disk. 
If **any** file path in the archive:
- Is an absolute path (e.g., `/etc/passwd`)
- Contains directory traversal components (e.g., `../`, `..\\`)
- Resolves to a location outside the designated output directory
Your program MUST immediately abort, print an error to standard error, and exit with status code `1` without writing any files.
Valid archives must be extracted successfully, exit with status code `0`, and have their files renamed with the prefix. Subdirectories should be created as needed.

### 4. Implementation Details
Initialize a new Rust binary project at `/home/user/pza_tool`.
Your CLI must accept exactly three arguments:
`cargo run --manifest-path /home/user/pza_tool/Cargo.toml -- <input_pza_path> <output_dir> <prefix>`

Once your tool is built, use it to safely process the archives provided in our testing corpora. You do not need to run the verifier yourself; an automated test will invoke your Rust binary against two distinct corpora of `.pza` files:
- A "clean" corpus of safe archives.
- An "evil" corpus of malicious archives.
Make sure your tool is fully compiled and ready at `/home/user/pza_tool`.
