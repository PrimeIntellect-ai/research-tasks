You are a technical writer managing a complex documentation compilation system. You have received an export file named `/home/user/input_docs.csv` generated by a third-party documentation tool. 

This CSV contains chunked, base64-encoded contents of various documentation files. However, a security scan revealed that the export tool is vulnerable to a "zip slip" style directory traversal attack: some `file_path` entries contain relative paths (like `../../`) or absolute paths meant to overwrite system files outside the intended extraction directory.

Your task is to write and execute a script (preferably in Bash, utilizing standard Linux tools) that processes this CSV and safely rebuilds the documentation.

Here are the requirements:
1. **Sanitize Paths**: Extract only the **basename** of the `file_path` for every row to prevent directory traversal. Discard any directory components (e.g., `../../../etc/passwd` becomes `passwd`, `chapter1/intro.md` becomes `intro.md`).
2. **Merge and Decode**: The files are split into multiple rows. For each unique sanitized file, you must reassemble it by ordering the chunks according to the `chunk_index` (which starts at 0 and increments). Concatenate the `content_base64` values for these chunks in order, and then decode the final concatenated base64 string into plain text.
3. **Atomic Writes**: Write the decoded plaintext to `/home/user/docs_output/<sanitized_basename>`. To prevent partial reads by other processes, you **must** use an atomic write pattern: write the content to a temporary file (e.g., `<sanitized_basename>.tmp`) in the same directory, and then use `mv` to atomically replace the target file.
4. **Manifest Generation**: Finally, create a JSON formatted manifest at `/home/user/docs_output/manifest.json`. The JSON should be a single dictionary mapping the sanitized basenames of the successfully created files to their final decoded character lengths (in bytes). 

Example manifest format:
```json
{
  "file1.md": 1024,
  "file2.txt": 42
}
```

**Constraints:**
* The CSV headers are exactly `file_path,chunk_index,content_base64`.
* The CSV is comma-separated; assume no commas exist within the `file_path` or `content_base64` fields.
* You must ensure the `/home/user/docs_output/` directory exists before starting.
