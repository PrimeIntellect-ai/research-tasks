You are an AI assistant helping a climate researcher organize and normalize an archive of legacy sensor datasets. 

The researcher has a compressed archive of raw logs and a configuration file. The logs were generated by older systems in different regions, meaning they are stored in various legacy character encodings. Furthermore, the log entries span multiple lines and contain both valid and invalid readings.

Here is your task:

1. **Environment Setup:**
   Navigate to `/home/user/research_data/`. You will find:
   - `config.ini`: A configuration file specifying the active stations, their corresponding log files, and their original character encodings.
   - `raw_logs.tar.gz`: A compressed archive containing the raw log files.

2. **Extraction and Parsing:**
   Extract `raw_logs.tar.gz`.
   Write a Python script that reads `config.ini` to determine which log files to process and what encoding to use for each. 
   
   Each log file contains multi-line records in the following format:
   ```
   ===RECORD START===
   STATION: <station_name>
   TIMESTAMP: <ISO-8601-datetime>
   STATUS: <VALID|INVALID|MAINTENANCE>
   NOTES: <Text in local encoding>
   ===RECORD END===
   ```
   *Note: There might be empty lines or garbage data between the `===RECORD END===` of one record and the `===RECORD START===` of the next.*

3. **Filtering and Conversion:**
   Extract ONLY the records where the `STATUS` is exactly `VALID`. 
   Convert all extracted text (especially the `NOTES` field) into UTF-8.
   
4. **Output Generation:**
   Format the valid records as a JSON Lines (`.jsonl`) file, where each line is a valid JSON object with the keys: `station`, `timestamp`, `status`, and `notes`.
   Save this file as `/home/user/research_data/normalized_logs.jsonl`.
   
5. **Archiving:**
   Compress `normalized_logs.jsonl` into a new ZIP archive located at `/home/user/research_data/export.zip`. Ensure the zip file contains just the `.jsonl` file at the root level (no directories).

Ensure your script handles the multi-line parsing carefully and properly decodes the text before creating the JSON objects.
