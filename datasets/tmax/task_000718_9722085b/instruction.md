You are a FinOps analyst tasked with optimizing cloud storage costs and ensuring the reliability of our internal cost-tracking services. You have two main objectives: fix a broken service deployment and create a file classification script to identify wasted storage.

**Part 1: Fix the Service Deployment**
In `/home/user/services`, we have a custom multi-service application managed by a basic shell script.
- `app_server.py`: A simulated data ingester. Upon startup, it performs some initialization and after 3 seconds, creates a directory structure and a symlink at `/home/user/app_data/active_data`.
- `cost_analyzer.py`: A daemon that monitors `/home/user/app_data/active_data` for storage metrics.
- `start_all.sh`: A supervisor script that starts both services in the background.

Currently, if you run `./start_all.sh`, the `cost_analyzer.py` service immediately crashes because it expects the `active_data` symlink to exist at launch. 
Your task is to modify `/home/user/services/start_all.sh` so that `cost_analyzer.py` correctly starts *only after* the symlink `/home/user/app_data/active_data` has been created by the `app_server.py`. Both processes must remain running indefinitely in the background when `./start_all.sh` is executed.

**Part 2: Build the Cost Cleanup Classifier**
To reduce our cloud storage bill, we need to aggressively delete temporary and orphaned files, but we must absolutely preserve active configuration and active data.

Write a Python script at `/home/user/classifier.py`.
The script must take a single command-line argument representing the absolute path to a file.
It must analyze the file and print exactly `DELETE` or `KEEP` to standard output.

Implement the following classification rules. The rules must be evaluated in this order:
1. If the file has a `.tmp` or `.bak` extension, print `DELETE`.
2. If the file has a `.config` extension, print `KEEP`.
3. Read the first line of the file. If it strictly starts with the exact string `[ORPHANED]`, print `DELETE`.
4. If the first line strictly starts with the exact string `[ACTIVE]`, print `KEEP`.
5. For all other files, default to printing `KEEP`.

You can test your classifier against the mock data generated by the services, but your final script will be rigorously tested against a hidden corpus of files. Ensure your Python script includes robust error handling (e.g., if a file is empty or unreadable, default to `KEEP`).
