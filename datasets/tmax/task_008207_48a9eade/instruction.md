You are an IT monitoring specialist. We need a custom, lightweight storage monitoring and alerting system written in Python that can monitor a specific directory, trigger a backup if it exceeds a quota, and serve its status securely via a local HTTPS API. 

Your objective is to create and run this system using a combination of a Bash deployment script and a Python monitoring daemon.

Here are the specific requirements:

1. **Deployment Script (`/home/user/setup_and_run.sh`)**
   Write a Bash script at `/home/user/setup_and_run.sh` that performs the following steps:
   - Creates necessary directories: `/home/user/certs` and `/home/user/backup_dir`.
   - Generates a self-signed RSA 2048-bit TLS certificate (`/home/user/certs/cert.pem`) and private key (`/home/user/certs/key.pem`) valid for 30 days with a Common Name (CN) of `localhost`.
   - Launches the Python monitoring daemon (described below) in the background.
   - Saves the daemon's Process ID (PID) into a file at `/home/user/monitor.pid`.
   - The script must be executable.

2. **Python Monitoring Daemon (`/home/user/monitor.py`)**
   Write a Python script at `/home/user/monitor.py` that does the following:
   - **Storage Monitoring & Backup:**
     Calculates the total size (in bytes) of all files inside `/home/user/data_dir`. 
     If the total size is strictly greater than 50,000,000 bytes (50 MB):
       a. It must create a compressed gzip tarball of `/home/user/data_dir` and save it to `/home/user/backup_dir/data_backup.tar.gz`.
       b. It must append exactly this string to a log file at `/home/user/alerts.log`: `ALERT: data_dir exceeds 50MB` (followed by a newline).
   - **Secure Web Server:**
     After performing the check and potential backup, the script must start an HTTPS server on `0.0.0.0` at port `8443`, using the `cert.pem` and `key.pem` generated by the bash script.
     The server must respond to `GET /status` requests with HTTP 200 and a JSON payload containing the current state.
     - If the directory exceeded the threshold, the JSON must be exactly: 
       `{"disk_status": "ALERT", "backup_path": "/home/user/backup_dir/data_backup.tar.gz"}`
     - If the directory was under the threshold, the JSON must be exactly: 
       `{"disk_status": "OK", "backup_path": null}`
     The server must run continuously.

3. **Execution**
   Once you have created the files, execute `/home/user/setup_and_run.sh` so the certificates are generated, the backup is created (since `/home/user/data_dir` currently contains a large dataset), and the secure web server begins listening on port 8443. 

Leave the web server running in the background. Ensure proper error handling in your scripts.
