As a backup administrator, you are tasked with archiving a set of custom log files generated by a legacy system. 

The logs are located in the directory `/home/user/logs`. The log rotation mechanism creates files named `log.0`, `log.1`, `log.2`, etc., and the current file being written to is named `active.log`. 

These log files use a custom binary framing format to prevent corruption during rotation races. Each log record consists of:
1. A 4-byte big-endian unsigned integer representing the length of the payload.
2. The payload itself, which is a UTF-8 encoded string containing comma-separated values (CSV) in the exact format: `timestamp,level,message`. There are no trailing newlines in the payload.

Your task is to write a Python script at `/home/user/archive_logs.py` that performs the following steps:
1. Discovers and reads all log files in `/home/user/logs`. The files must be processed in strict chronological order: `log.0` first, followed by `log.1`, `log.2`, etc., and finally `active.log`.
2. Parses the binary framing to extract each CSV payload.
3. Converts the extracted CSV payloads into a structured JSON object with the keys `"timestamp"`, `"level"`, and `"message"`.
4. Writes the resulting JSON objects as a JSON Lines (JSONL) stream directly into a gzip-compressed archive located at `/home/user/archive.json.gz`.

Once you have written the script, execute it to produce the final `/home/user/archive.json.gz` file.
