You are tasked with writing a robust configuration extraction script for our new server management system. Our system receives configuration updates as tar archives, but we have discovered that some archives contain malicious paths attempting a "tar-slip" (or zip-slip) attack by writing files outside the intended target directory.

Additionally, our release process embeds a deployment identifier in a video artifact generated by our CI/CD pipeline, and our configuration files need to be standardized (encoding and comments) and safely extracted using file locks to prevent race conditions during concurrent deployments.

Write a bash script at `/home/user/safe_extract.sh` that takes exactly two arguments:
1: `ARCHIVE` (path to a `.tar` file)
2: `DEST` (path to the target extraction directory)

Your script must perform the following strictly in order:

**1. File Locking:**
To prevent concurrent runs from corrupting the directory, your script must obtain an exclusive file lock on a lockfile named `.config.lock` inside the `DEST` directory (create the directory if it doesn't exist). Use file descriptor `9` for this lock via `flock`. Hold the lock for the entire duration of the script's execution.

**2. Video Analysis for Deployment ID:**
Analyze the video file located at `/app/deploy_logs.mp4`. You must count the number of "scene changes" in this video. A scene change is defined as a frame where the `scene` score is strictly greater than `0.4` (using `ffmpeg`'s `select` filter). Let this integer count be `N`.

**3. Safe Extraction & Bulk Renaming:**
Read the files inside the `ARCHIVE`. For each file:
* Evaluate its extraction path relative to `DEST`. 
* If the archive path attempts to escape the target directory (e.g., contains `../` sequences that resolve above `DEST` or is an absolute path starting with `/`), it must NOT be extracted to its intended location. Instead, it must be extracted into a quarantine directory at `DEST/quarantine/`.
* For ALL files (both normal and quarantined), you must rename the extracted file (the basename, not the directories) according to these rules:
  - Convert all uppercase letters to lowercase.
  - Replace all spaces with underscores (`_`).
  - Prefix the basename with `deploy_${N}_` (where `${N}` is the scene change count calculated in step 2).
*(Example: if `N` is 3, and a valid archive file is `config/My App Settings.conf`, it should be extracted as `DEST/config/deploy_3_my_app_settings.conf`. If it was `../../../etc/Evil File.sh`, it becomes `DEST/quarantine/deploy_3_evil_file.sh`)*.

**4. Encoding Conversion & Text Transformation:**
After extracting and renaming, scan all the newly extracted files. For any file whose final name ends in `.conf`:
* Convert its character encoding from `ISO-8859-1` to `UTF-8`.
* Use `sed` (or `awk`) to remove any line that begins with a `#` character (ignoring any leading whitespace). The file must be modified in-place.

Your script must be executable (`chmod +x`). Ensure all operations within the script rely exclusively on standard bash tools, `tar`, `sed`, `iconv`, `flock`, and `ffmpeg`.
