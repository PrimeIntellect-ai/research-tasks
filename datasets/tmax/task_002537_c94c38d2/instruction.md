You are an advanced security AI assisting a defensive team in building a robust file upload WAF (Web Application Firewall) module. A previous red-team engagement successfully bypassed our old file upload handler using path traversal (CWE-22) and encrypted payloads. We have a screen recording of their terminal session during the attack.

Your objective is to extract the adversary's encryption key from the recording, and then develop a C++ sanitization program that correctly identifies and blocks their techniques while allowing legitimate traffic.

**Stage 1: Intelligence Extraction**
1. An MP4 recording of the attacker's terminal is located at `/app/session.mp4`.
2. Analyze the video (e.g., by extracting frames using `ffmpeg`) to find the exact AES-256-CBC encryption key the attacker used to encrypt their payloads. It is clearly typed on the screen in one of the commands.

**Stage 2: C++ Sanitizer Development**
Create a C++ program at `/home/user/sanitizer.cpp` and compile it to `/home/user/sanitizer`.
This program must take exactly one argument: the absolute path to a payload metadata file.

The payload metadata file is a plain text file with the following format:
```
FILENAME: <string>
SHA256: <hex_string>
PAYLOAD: <base64_encoded_string>
```

Your `sanitizer` must parse this file and determine if it is CLEAN or EVIL based on the following rules:
1. **CWE-22 (Path Traversal):** If the `FILENAME` contains sequences designed to navigate directories (e.g., `../`, `..\\`, or URL-encoded variants like `%2e%2e%2f`), it is EVIL.
2. **File Integrity:** Compute the SHA256 hash of the decoded binary data in the `PAYLOAD` field. If it does not match the `SHA256` field exactly, it is EVIL (tampered).
3. **Encrypted Malicious Content:** Attempt to decrypt the decoded binary `PAYLOAD` using AES-256-CBC and the key you extracted from the video. The IV is always an all-zero 16-byte array. If the decryption succeeds (valid padding) AND the decrypted plaintext contains the exact string `REDTEAM_STRIKE`, it is EVIL.
4. If it passes all the above checks, it is CLEAN.

**Program Output Requirements:**
- If the file is EVIL, print `EVIL` to standard output and exit with status code `1`.
- If the file is CLEAN, print `CLEAN` to standard output and exit with status code `0`.

**Stage 3: Verification**
We have provided two testing directories:
- `/app/corpus/clean/` contains benign payload files.
- `/app/corpus/evil/` contains malicious payload files generated by the red team.

You must ensure your `/home/user/sanitizer` correctly flags 100% of the files in `/app/corpus/evil/` as EVIL and 100% of the files in `/app/corpus/clean/` as CLEAN. You can test your compiled program against these directories before finalizing your solution.
