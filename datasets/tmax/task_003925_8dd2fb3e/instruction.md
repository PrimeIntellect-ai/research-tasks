You are a data scientist debugging a Bash-based data cleaning pipeline for a legacy system. The system consists of two bash-based microservices located in `/app/pipeline/`:

1.  **Ingest Service** (`ingest.sh`): Listens on HTTP port 8080. It accepts POST requests containing raw CSV data and streams it to the processing script (`process.sh`).
2.  **API Service** (`api.sh`): Listens on HTTP port 8081. It serves the current statistics generated by the pipeline as a JSON response.

Currently, the pipeline crashes or returns garbage data because the `process.sh` script is incomplete and mishandles malformed CSVs. Your task is to fix and implement `/app/pipeline/process.sh` using only Bash, `awk`, `sed`, `grep`, and coreutils. No Python or Perl allowed.

### Requirements for `process.sh`:
The script receives CSV data via standard input. The CSV has four columns: `timestamp,category,value,notes`.
1.  **Data Cleaning:** Silently drop any logical CSV rows that contain embedded newlines. In our dataset, a field with an embedded newline is enclosed in double quotes (`"`). You must detect lines with unbalanced quotes (an odd number of `"` characters) and skip them, as well as any subsequent lines, until the quote is closed. The entire multi-line logical row must be dropped.
2.  **Sorting:** Sort the remaining valid, single-line rows numerically by the `timestamp` column (column 1).
3.  **Rolling Statistics:** For the valid, sorted rows, compute a rolling sum of the `value` column (column 3) using a window size of exactly 3 (i.e., the current row's value plus the previous two rows' values). For the first two rows, the rolling sum is just the sum of the available rows up to that point.
4.  **Template Generation:** Output the final statistics to the file `/tmp/stats.json` in the following exact JSON format (replacing `<COUNT>` with the total valid rows, and the array with the rolling sums):
    ```json
    {
      "valid_rows": <COUNT>,
      "rolling_sums": [ <SUM1>, <SUM2>, ... ]
    }
    ```

### Services Configuration:
*   You must start the services using `/app/pipeline/start.sh`.
*   Ensure that `api.sh` responds to `GET /stats` requests on `127.0.0.1:8081` by reading and returning the contents of `/tmp/stats.json`. You may need to edit `api.sh` to ensure it returns the correct HTTP headers and the JSON file content.
*   Do not change the ports.

Fix the scripts, start the services, and verify that the pipeline correctly processes incoming data.
