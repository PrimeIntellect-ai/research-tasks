You are a data analyst tasked with processing messy daily reports that contain embedded time series data. The system currently drops raw report files into `/home/user/data/raw/`. These files are generated by different legacy systems, meaning they come in various character encodings (e.g., UTF-8, UTF-16, ISO-8859-1) and contain unstructured text.

Your objective is to build a robust data pipeline that extracts the time series data and schedules its processing. 

Step 1: Information Extraction & Encoding Handling (Python)
Write a Python script at `/home/user/process.py` that takes a single file path as a command-line argument.
- The script must read the file, correctly handling its character encoding. You may need to try multiple common encodings (utf-8, utf-16, iso-8859-1) or use a heuristic approach to decode the text successfully without throwing errors.
- Extract the timestamp and metric value from the text. The data is always embedded in the format: `Readout: [YYYY-MM-DD HH:MM:SS] -> <value> units.` (e.g., `Readout: [2023-10-01 10:00:00] -> 15.2 units.`).
- The script should print a single CSV formatted line to standard output: `YYYY-MM-DD HH:MM:SS,value`.

Step 2: Pipeline Creation (Bash)
Write a bash script at `/home/user/pipeline.sh` that:
- Iterates through all `.dat` files in `/home/user/data/raw/`.
- Executes `/home/user/process.py` on each file.
- Accumulates the results into `/home/user/data/timeseries.csv`.
- Ensures that `/home/user/data/timeseries.csv` has exactly one header row: `timestamp,value`.
- Sorts the data rows chronologically by timestamp in the final CSV (header must remain at the top).

Step 3: Scheduling
We need to run this pipeline automatically. Create a cron job configuration file at `/home/user/pipeline_cron` that schedules `/home/user/pipeline.sh` to execute every day at exactly 2:30 AM. The file should contain just the single valid crontab line required for this schedule.

Before finishing, run your pipeline script once manually so that `/home/user/data/timeseries.csv` is populated with the data currently in `/home/user/data/raw/`.

Make sure your scripts are executable (`chmod +x`). All file paths must be exactly as specified.
