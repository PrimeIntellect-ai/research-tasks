You are a data scientist tasked with cleaning up a dataset generated by a flaky ETL job. Because the job frequently retries on failure, the output contains duplicate records.

Write an executable Bash script at `/home/user/clean_etl.sh` that reads a raw CSV data stream from standard input (stdin) and writes the cleaned, deduplicated, and normalized data to standard output (stdout).

The input stream has no header. Each line contains four comma-separated columns:
`Timestamp,UserID,Event,Value`

Your script must perform the following pipeline operations:
1. **Filtering**: A colleague left a screenshot of the internal filtering rules at `/app/rules.png`. You must use OCR or an image viewer to read the text in this image and drop any records matching the banned event specified in the image.
2. **Normalization**: Convert the `Event` column entirely to UPPERCASE. Format the `Value` column to exactly two decimal places (e.g., `42.1` becomes `42.10`, `15` becomes `15.00`).
3. **Deduplication**: Deduplicate the stream by `UserID`. For each user, keep ONLY the record with the strictly highest `Timestamp` (which is an integer). If multiple records for the same `UserID` have the exact same maximum timestamp, keep the one that appeared *latest* (last) in the input stream.
4. **Formatting and Sorting**: Output the cleaned records in the format: `UserID,Timestamp,Event,Value`. Notice the column reordering! Finally, sort the output alphabetically by `UserID`.

Your script must be robust enough to handle millions of rows via streaming without buffering the entire file if possible (though buffering the deduplicated records in memory is acceptable). Ensure the script is executable (`chmod +x`). Do not print any extraneous text or debug information to standard output, as it will be programmatically tested against a reference implementation using randomly generated streams.
