You are a systems programmer debugging a C library linking issue within a hybrid Go/C project, and you need to build a testing harness in Bash to verify the program's output.

The project is located at `/home/user/project`. It consists of:
1. `asmgen.c` and `asmgen.h`: A C library that generates custom assembly-like instructions.
2. `main.go`: A Go program that concurrently calls the C library using goroutines to generate 5 assembly files (`output_1.asm` to `output_5.asm`).
3. `Makefile`: A build script that currently fails due to C library linking and compilation errors.

Your tasks are to:
1. **Fix the Build**: Edit the `Makefile` (and optionally `main.go` if needed) to correctly compile `libasmgen.so` as a shared library and link it with the Go program to produce the `generator` executable.
2. **Implement an Interpreter**: Write a Bash script at `/home/user/interpreter.sh` that implements a simple stack machine interpreter to execute the `.asm` files generated by the C library. 
   The interpreter must process the following commands line-by-line:
   - `PUSH <value>`: Pushes an integer onto the top of the stack.
   - `ADD`: Pops the top two values, adds them, and pushes the result.
   - `SUB`: Pops the top value (A) and the next value (B), computes B - A, and pushes the result.
   - `MUL`: Pops the top two values, multiplies them, and pushes the result.
   - `PRINT`: Pops the top value and prints it to standard output.
3. **Write a Test Harness**: Write a Bash script at `/home/user/run_tests.sh` that does the following:
   - Cleans and builds the project using `make`.
   - Runs the `./generator` executable (ensure the runtime linker can find `libasmgen.so`).
   - Uses `/home/user/interpreter.sh` to execute the generated `output_1.asm` through `output_5.asm` in order.
   - Appends the output of each interpreter run to `/home/user/test_results.log`.

Make sure `/home/user/run_tests.sh` and `/home/user/interpreter.sh` are executable. Do not change the logic in `asmgen.c`.
