Our pipeline uses a Jupyter notebook (`/home/user/fit_model.ipynb`) to perform gradient-descent optimization to extract peak locations from spectroscopy data. Because we have thousands of output files, we orchestrate the notebook execution using `papermill`. 

However, we are facing two major issues that you must resolve:

1. **Broken Orchestration Tool**: Because of strict security constraints, we cannot install tools from the internet. We have vendored the source code for `papermill` version 2.3.4 at `/app/vendored/papermill-2.3.4`. Unfortunately, a previous developer made a botched patch to it, and it currently fails to run. You must find the syntax error or broken environment assumption in its CLI module, fix it, and install the package into the system environment (e.g., `pip install -e /app/vendored/papermill-2.3.4`).

2. **Simulation Data Anomalies**: The upstream simulation runs across many threads. Sometimes, floating-point reduction order issues cause non-reproducible sequences in the output. Specifically, the simulation output files (two-column CSVs without headers: `Wavelength, Intensity`) occasionally contain `NaN` or `Inf` values in the intensity, or the `Wavelength` values (first column) are printed *out of order* (they should be strictly monotonically increasing). When these anomalous files are fed into the notebook, the area-under-curve and optimization steps produce wild, non-reproducible results.

Your task is to:
1. **Fix and install the vendored `papermill` package**.
2. **Build a Bash Classifier**: Create an executable Bash script at `/home/user/detect_bad_spectrum.sh`. It must accept exactly one argument (the path to a CSV file). 
   - If the file is perfectly valid (strictly monotonically increasing Wavelengths, no `NaN`/`Inf`/`nan`/`inf` anywhere), the script must print `VALID` to stdout and exit with code `0`.
   - If the file contains out-of-order wavelengths or any NaN/Inf values, the script must print `INVALID` to stdout and exit with code `1`.
3. **Build the Orchestrator**: Create a Bash script at `/home/user/orchestrate.sh` that takes two arguments: `<input_dir>` and `<output_dir>`. 
   - It should iterate over all `*.csv` files in `<input_dir>`.
   - It must call `detect_bad_spectrum.sh` on each file.
   - For every file that is `VALID`, it must use the fixed `papermill` to execute `/home/user/fit_model.ipynb`, saving the output notebook to `<output_dir>/<basename_of_csv>_out.ipynb`. It must pass the parameter `input_file` to the notebook pointing to the CSV path.
   - For `INVALID` files, it should skip notebook execution.

For your testing, there are two sample directories:
- `/app/corpora/clean/` contains correctly ordered, well-formed spectra.
- `/app/corpora/evil/` contains anomalies (unsorted rows, NaNs, etc.).

Ensure both of your Bash scripts are executable. Your solution will be tested extensively against hidden datasets.
